# 📚 Q-Net Study

> Q-Net(큐넷) 자격증 기출을 **브라우저에서 풀고, 자동 채점·기록하고, 오답을 관리**하는 로컬 웹 앱 + 스터디 저장소입니다.
> 서버는 내 컴퓨터에서만 돌고(127.0.0.1 전용), 모든 기록은 마크다운 파일로 남아 git으로 스터디원과 공유합니다.

## 🚀 빠른 시작 (3분)

**1. 준비물** — [Node.js](https://nodejs.org/ko) 18 이상만 있으면 됩니다. (`node -v`로 확인)

**2. 클론 & 실행**

```bash
git clone https://github.com/hun-meta/Q-Net-Study.git
cd Q-Net-Study/web
npm install
npm start
```

**3. 접속** — 터미널 배너에 뜨는 주소(`http://127.0.0.1:4525`)를 브라우저로 열고 → **닉네임 선택**. 끝!

> 포트가 사용 중이면 4526~4535로 자동 폴백됩니다. 배너의 주소를 확인하세요.

### 선택 준비물 — AI 기능 (없어도 핵심 기능은 전부 동작)

| 도구 | 있으면 되는 것 | 없으면 |
|------|----------------|--------|
| [Claude Code](https://claude.com/claude-code) (`claude`) 설치+로그인 | 기출 업로드 시 **정답 자동 추출**, 학습 대화 **노트 자동 정리**, **마이크로월드 생성** | 정답을 수동 입력 폼으로 등록 (막다른 길 없음) |
| antigravity (`agy`) 설치+로그인 | 문항별 **AI 챗 질의응답** | 챗 탭만 비활성 |

미설치면 상단 배너로 알려주며, 나중에 설치하면 서버가 30초 내 자동 감지해 활성화합니다.

## 🖥️ 웹에서 할 수 있는 것

```
자격증 등록 → 기출 PDF 업로드(정답 자동 추출) → 풀이(PDF 뷰어 + OMR)
   → 제출·자동 채점(과목 점수·과락·합격) → 결과·추이 → 오답 키워드 정리
   → 개념 보기(내·스터디원 노트 + 공유 해설) → AI 챗 → 승인하면 노트에 자동 기록
```

- **자격증 등록**: 사이드바 "＋ 자격증 등록" — 분야/자격증명만 입력하면 공유 폴더 골격이 생깁니다.
- **기출 업로드**: 자격증 화면에 PDF를 드롭하면 서버가 배치하고, `claude`가 답지를 판독해 정답을 등록합니다. 실패하면 그 이유가 표시되고 수동 입력 폼이 열립니다.
- **풀이 화면**: 왼쪽 PDF 뷰어(답지 페이지는 서버가 제거한 버전만 전송 — 스포일러 차단), 오른쪽 OMR. 각각 독립 스크롤, 키보드 입력(1~4, ↑↓, G=찍음) 지원. 입력은 자동 임시저장돼 나중에 이어 풀 수 있습니다.
- **확신도가 핵심**: 모르고 찍은 문항은 OMR에서 "찍음"을 체크하세요. **맞았어도 찍은 문제는 오답 인덱스에 들어가** 복습 대상이 됩니다. 우선순위: 🔴 틀림+찍음 → 🟠 틀림+애매 → ⛔ 틀림+확신(위험한 오개념!) → 🟡 맞음+찍음.
- **개념 보기**: 문항마다 연결된 내 노트·스터디원 노트·공유 해설을 한 화면에서 봅니다.
- **챗 → 승인 정리**: 문항에 대해 AI와 대화하고, "정리해줘"를 누르면 대화 내용이 저장소 규칙대로 내 노트/공유 해설에 기록됩니다(승인 전엔 아무 파일도 쓰지 않음).
- **🌐 마이크로월드**: 자격증 화면에서 개념(예: CPU 스케줄링)을 입력하면 직접 조작하며 이해하는 인터랙티브 시뮬레이션을 생성·체험합니다. 생성물은 스터디원 모두가 공유.

## 📂 내 기록은 어디에 남나

DB가 없습니다. **모든 기록이 사람이 읽을 수 있는 마크다운 파일**로 저장소 안에 남습니다:

```
정보처리/정보처리기사/
├── _공통/            🤝 스터디원 공유 (기출 PDF·정답·출제기준·해설·마이크로월드)
└── {내닉네임}/        👤 내 공간 (노트 notes/ + 풀이 기록 attempts/)
```

- `_공통/`은 누구나 수정 OK. **남의 닉네임 폴더는 보기만, 수정 금지.**
- 풀이 기록·오답 인덱스(`WRONG.md`)·점수 추이가 전부 여기 쌓이므로, git으로 커밋·푸시하면 스터디원과 자연스럽게 공유됩니다.

## 🤝 스터디원과 공유하기

기록을 원격(GitHub)과 동기화하려면 git을 직접 쓰거나, Claude Code가 있다면 한 마디로:

```
/sync-origin     ← 미커밋 자동 커밋 → 원격 가져오기 → 충돌 조정 → push
```

규칙은 두 가지뿐: ① 남의 닉네임 폴더를 수정하지 않는다 ② 문제집 스캔본 등 저작권 자료는 올리지 않는다(Q-Net 공개 기출·자작 복원은 OK).

| 기동 실패(포트) | 4525~4535 모두 점유됨. 다른 서버 종료 후 재시도 |

## 🧠 Deep 사용자용 — Claude Code와 직접 쓰기

웹 앱 없이(또는 병행해서) Claude Code 대화로 스터디 전체를 운영할 수 있습니다. 저장소를 Claude Code로 열면 아래 스킬이 활성화됩니다.

> ⚠️ 병행 시 주의: 웹 앱에서 추출·정리 같은 AI 잡이 **실행 중일 때** Claude Code로 저장소 파일을 수정하면, 잡의 사후 감사가 이를 경계 위반으로 오인해 잡 결과와 그 수정을 함께 원복합니다. 잡이 끝난 뒤 편집하세요.

### 학습 사이클 스킬

```
/cert-init 정보처리기사 홍길동     ← 자격증 조사 + _공통/ 구축 + 개인 공간 생성
① /study-note   개념 정리       "소프트웨어설계 과목 정리해줘"
② /exam-pdf     기출 PDF 등록    "이 PDF 기출문제로 등록해줘"
③ /attempt-log  풀고 채점·기록   "2023년 1회 필기 풀었어, 채점해줘"
④ /review       오답 복습        "오답 복습 시작"  →  다시 ①로
```

| 스킬 | 하는 일 |
|------|---------|
| `/cert-init` | 자격증 정보 조사 → `_공통/`(info·출제기준·기출 인덱스) 구축 → 닉네임 공간 생성 |
| `/study-note` | **출제기준 계층 그대로** 개념 노트 작성·보강 (임의 목차 금지) |
| `/exam-pdf` | PDF 명명 규칙 적용·판독·인덱스 등록, 웹 앱과 같은 정답 md 포맷 |
| `/attempt-log` | 채점 + 풀이 기록 + `INDEX.md`/`WRONG.md` 자동 갱신 |
| `/review` | 오답 우선순위대로 재출제·재채점, 확신 2연속 정답 시 졸업 처리 |
| `/sync-origin` | main 전용 원격 동기화(자동 커밋 → fetch → 리베이스 충돌 조정 → push) |

### 알아두면 좋은 규칙 요약

- **이름 규칙**: 기출 PDF `연도-회차-구분.pdf`(예: `2023-1-필기.pdf`) / 풀이 기록 `연도-회차-구분-시도.md` / 노트 `순번-주요항목.md`(예: `01-요구사항확인.md`)
- **확신도 3단계**: `확신`(근거 알고 선택) / `애매`(고민) / `찍음`(모름) — 채점보다 중요한 실력 지표
- **공유 해설**(`_공통/풀이/`): 한 파일에 여러 명이 기여하되 각자 `## 닉네임 (날짜)` 서명 섹션에만 append. 남의 섹션은 불가침
- **커밋 메시지**: `[자격증/닉네임] 요약` (공통 자료는 `[자격증/공통] 요약`)
- 전체 규칙은 **[CLAUDE.md](CLAUDE.md)** (Claude가 따르는 규칙이지만 사람이 읽어도 유용), 양식은 **[templates/](templates/)**, Q-Net 자격 체계 조사는 **[docs/qnet-guide.md](docs/qnet-guide.md)** 참조
