# 📚 Q-Net Study

> Q-Net(큐넷) 자격증을 **여러 명이 함께** 공부하고 기록하는 저장소입니다.
> 개념 정리 → 기출 풀이 → 오답 복습까지, Claude Code 스킬로 반자동 관리됩니다.

## 🚀 30초 시작하기

Claude Code를 열고 아래처럼 말하면 끝입니다:

```
/cert-init 정보처리기사 홍길동
```

→ 자격증 정보를 조사해서 공유 자료(`_공통/`)를 만들고, 내 이름의 개인 공간을 만들어 줍니다.

그다음은 이 사이클을 반복하면 됩니다:

```
① /study-note   개념 정리      "소프트웨어설계 과목 정리해줘"
② /exam-pdf     기출 PDF 등록   "이 PDF 기출문제로 등록해줘"
③ /attempt-log  풀고 채점·기록  "2023년 1회 필기 풀었어, 채점해줘"
④ /review       오답 복습       "오답 복습 시작"  →  다시 ①로
```

## 📁 디렉토리 구조

```
{종류}/{자격증}/{닉네임}/…   ← 기본 규칙
```

실제 모습:

```
정보처리/                       📂 분야 (같은 분야의 여러 등급을 묶음)
└── 정보처리기사/
    ├── _공통/                 🤝 모두가 함께 쓰는 공간
    │   ├── info.md               시험 개요·과목·일정·합격 기준·등급
    │   ├── 출제기준/              과목별 출제기준 (노트 목차의 기준!)
    │   └── 기출문제/              기출 PDF + INDEX.md
    ├── 홍길동/                 👤 홍길동만 쓰는 공간
    │   ├── README.md             내 학습 현황판
    │   ├── notes/                개념 정리 노트
    │   └── attempts/             기출 풀이 기록 + 오답 인덱스
    └── 김철수/                 👤 김철수만 쓰는 공간
```

- **종류** = **분야**(직무·주제, 예: `정보처리` `전기` `안전관리`). 이름은 자유(단 `web`·`docs`·`templates`·`node_modules`·`.`/`_` 시작은 예약).
- 자격 **등급**(기사/산업기사 등)은 각 자격증의 `_공통/info.md`에 메타데이터로 기록합니다.
- `_공통/`은 누구나 수정 OK. **남의 닉네임 폴더는 보기만, 수정 금지!**

## 🛠️ 스킬 안내

| 스킬 | 언제 쓰나 | 하는 일 |
|------|----------|---------|
| `/cert-init` | 스터디 시작할 때 | 자격증 조사 + 공유 자료 + 개인 공간 생성 |
| `/study-note` | 개념 공부할 때 | 출제기준 순서 그대로 노트 작성·보강 |
| `/exam-pdf` | 기출 PDF 구했을 때 | 이름 규칙 적용, 내용 판독, 인덱스 등록 |
| `/attempt-log` | 기출 풀었을 때 | 채점 + 풀이 기록 + 오답 인덱스 자동 갱신 |
| `/review` | 시험 전 복습할 때 | 오답 우선순위대로 재출제, 졸업 관리 |

## ✍️ 풀이 기록은 이렇게 남습니다

문항마다 **내 답 / 정답 / 결과 / 확신도**를 기록합니다:

| # | 내 답 | 정답 | 결과 | 확신도 | 메모 |
|---|------|------|------|--------|------|
| 1 | ③ | ③ | O | 확신 | |
| 2 | ② | ④ | X | 찍음 | 정규화 개념 부족 |

**확신도 3단계** — 채점보다 중요한 진짜 실력 지표입니다:

- `확신` : 근거를 알고 골랐다
- `애매` : 둘 중에 고민했다
- `찍음` : 모르겠어서 찍었다

맞았어도 찍은 문제는 오답 인덱스(`WRONG.md`)에 들어갑니다. 복습 우선순위:

> 🔴 틀림+찍음 → 🟠 틀림+애매 → ⛔ 틀림+확신(위험한 오개념!) → 🟡 맞음+찍음 → ⚪ 맞음+애매

`/review`에서 **확신 있게 2회 연속 정답**이면 그 문항은 졸업(취소선) 처리됩니다.

## 📏 이름 규칙 (외울 건 이것뿐)

| 대상 | 규칙 | 예시 |
|------|------|------|
| 기출 PDF | `연도-회차-구분.pdf` | `2023-1-필기.pdf` |
| 풀이 기록 | `연도-회차-구분-시도.md` | `2023-1-필기-1.md` |
| 개념 노트 | `순번-주요항목.md` | `01-요구사항확인.md` |

## 🤝 함께 쓰는 규칙

1. 남의 닉네임 폴더는 수정하지 않기 (참고는 자유!)
2. `_공통/`에 좋은 자료를 발견하면 적극 공유하기
3. 커밋 메시지: `[정보처리기사/홍길동] 소프트웨어설계 01 노트 작성`
4. 문제집 스캔본 등 저작권 있는 자료는 올리지 않기 (공개 기출은 OK)

## 🖥️ 기출 풀이 웹 앱 (로컬 실행)

기출 PDF를 브라우저에서 풀고 자동 채점·기록하는 로컬 웹 앱이 `web/`에 제공됩니다(개발 진행 중).

**⚠️ 실행 전제 조건 (필수)**

| 도구 | 용도 | 확인 방법 |
|------|------|----------|
| Node.js | 서버 실행 | `node -v` |
| **Claude Code** (`claude`) — 설치 + 로그인 | 정리·기록, 정답 추출 | `claude --version` |
| **antigravity** (`agy`) — 설치 + 로그인 | 챗 질의응답 답변 | `agy --version` |

두 CLI는 서버가 `claude --dangerously-skip-permissions` / `agy --dangerously-skip-permissions`로 호출합니다. 미설치·미로그인 상태면 해당 기능(챗/자동 정리·추출)이 비활성화되며, 풀이·채점·기록 열람은 계속 동작합니다.

실행:

```bash
cd web
npm install && npm start   # 127.0.0.1:4525 접속 → 닉네임 선택
```

**사용 플로우 (요약)**

```
닉네임 선택 → 기출 업로드/선택(정답 자동 추출·INDEX 부기) → 풀이(뷰어+OMR, 자동 임시저장)
  → 제출·채점(과목 점수·과락·합격) → 키워드 폼 → 개념 보기(🔁 태그: 노트+공유 해설)
  → 챗(agy, 표시만) → 승인 정리(claude가 목적지 선택대로 md 직접 기록)
```

- 답지 페이지는 서버가 제거한 PDF 서브셋만 전송하고, 정답 미등록 기출은 "열람만"으로 표시됩니다.
- `claude`/`agy` 잡은 **사후 감사**(경계 위반 원복·공유 해설 서명 보호·설정/훅 해시 감시)로 통제됩니다.
