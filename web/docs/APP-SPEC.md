# Q-Net 기출 풀이 웹 앱 — 기능·플로우·사용법 전체 명세

> **목적**: 이 문서만 읽고 모든 화면을 디자인할 수 있게 하는 디자인 핸드오프 명세.
> 기준일 2026-07-21. `[R2]` 표시 = 기능 라운드 2에서 구현 중(계약 확정, UI 포함).
> 함께 볼 것: `web/docs/ui-ux-round.md` (디자인 요구·제약), `web/README.md` (실행·트러블슈팅)

## 1. 앱 개요

- **한 줄**: Q-Net 자격증 기출 PDF를 브라우저에서 풀고, 자동 채점받고, 오답을 개념 노트·AI 챗으로 복습하는 **로컬 전용** 웹 앱.
- **사용자**: 스터디원 각자가 자기 컴퓨터에서 실행 (서버 배포 없음, 127.0.0.1 전용, 인증 없음). 데이터는 git 저장소의 md/PDF 파일 — 커밋·푸시로 스터디원과 공유.
- **실행**: `cd web && npm install && npm start` → `http://127.0.0.1:4525` (점유 시 4526~4535).
- **전제 조건**: Node.js + Claude Code(`claude`, 정리·정답추출) + antigravity(`agy`, 챗) 설치·로그인. **CLI가 없어도 풀이·채점·기록·열람은 전부 동작** — 챗·자동추출·승인정리만 비활성.
- **핵심 가치 루프**: 기출 풀기 → 채점 → 오답·찍은 문항 파악 → 개념 보기/챗으로 학습 → 승인한 내용만 노트에 기록 → 재풀이로 추이 확인.

## 2. 데이터 구조 (화면이 다루는 대상)

저장소 디렉토리 = 화면의 탐색 계층:

```
{종류}/{자격증}/                    예: 정보처리/정보처리기사/
├── _공통/                          (스터디 공유)
│   ├── info.md                     시험 개요·합격 기준
│   ├── 출제기준/{과목}.md           개념 섹션의 목차 원천
│   ├── 기출문제/
│   │   ├── INDEX.md                기출 목록 원천 (연도·회차·구분·문항수·정답포함·숨김페이지수)
│   │   ├── {연도}-{회차}-{구분}.pdf  예: 2023-1-필기.pdf
│   │   └── 정답/{시험ID}.md         채점 원천 (없으면 "열람만")
│   └── 풀이/{시험ID}/{문번}.md      문항별 공유 해설 (닉네임 서명 섹션)
└── {닉네임}/
    ├── notes/{과목}/{NN}-{주요항목}.md   개념 노트 (🔁 태그로 문항과 연결)
    └── attempts/                        풀이 기록·INDEX·WRONG (제출 시 자동 생성)
```

- **종류 = 분야** (정보처리, 전기, 안전관리…) — 자유 명명. `[R2]`로 등급제(기사/산업기사)에서 전환됨.
- **참여자 레지스트리** `[R2]`: 루트 `참여자.md` — 닉네임 등록 즉시 모든 화면 드롭다운에 반영.
- **시험ID**: `{연도}-{식별자}-{구분}` (예: `2023-1-필기`, `2024-상시-2-필기`). 구분은 필기/실기.

## 4. 화면 플로우 (전체 지도)

```
[온보딩: 닉네임]
   └─ 선택/등록 완료 →
[홈: 종류·자격증 탐색]  ←→  [자격증 신규 등록 R2]
   └─ 자격증 선택 →
[기출 목록 + 업로드]
   ├─ 기출 선택 → [풀이 화면] ─ 제출 → [채점 결과] ─→ [키워드 폼] ─→ (개념/챗으로 복습)
   ├─ PDF 업로드 → (자동 추출) → 목록 갱신 / 실패 시 [수동 정답 입력 폼]
   └─ 이력 보기 → [시도 이력·추이]
[풀이/결과 화면 내 도구]
   ├─ 문항별 [개념 보기 패널]
   └─ 문항별 [챗 패널] ─ "정리해줘" → [승인 다이얼로그] → AI 기록 → 개념 보기에 반영
```

## 5. 화면별 상세

### 5.1 온보딩 — 닉네임 `[R2로 개편]`
- **트리거**: 닉네임 미설정 상태로 접속 시 (첫 실행).
- **구성**: 기존 참여자 **드롭다운**(참여자.md ∪ 디렉토리 스캔 합집합) + "새 닉네임 등록…" 선택 시 텍스트 입력으로 전환 → 저장.
- **검증 에러**: 빈 값/40자 초과/경로 문자/`.` 시작/`_공통` 예약어 — 인라인 에러 문구.
- **닉네임 설정 후**: 상단에 항상 표시, [변경](드롭다운 재표시) / [삭제] 버튼.
- **삭제 플로우** `[R2]`: [삭제] → 서버에서 삭제 영향 조회(`usage`: 삭제될 디렉토리 목록+파일 수) → **확인 다이얼로그**: 경고문("모든 자격증의 개인 디렉토리가 삭제됩니다. git 커밋 전 기록은 복구 불가") + 삭제 대상 목록 + **닉네임을 직접 입력해야 활성화되는 위험 버튼** → 성공 시 삭제된 경로 결과 표시. 현재 닉네임을 삭제했다면 온보딩으로 복귀.
- 참고: 공유 해설(_공통/풀이/)의 그 사람 서명 섹션은 **보존**됨(공유 자산) — 다이얼로그에 명시.

### 5.2 홈 — 종류·자격증 탐색
- **데이터**: 종류(분야) → 자격증 목록(참여자 수, _공통 유무).
- **현재 구현**: 사이드바 셀렉트("종류 / 자격증"). 디자인 재구성 자유(대시보드 카드형 권장 — ui-ux-round.md 참고안).
- **자격증 신규 등록** `[R2]`: 종류 입력(기존 종류 datalist 제안) + 자격증명 → 생성(_공통 골격 자동) → 목록 갱신·자동 선택. 에러: 중복(409 "이미 존재"), 이름 검증 실패(400).

### 5.3 기출 목록 (자격증 선택 후)
- **데이터 행**: 시험ID(연도·회차·구분), 문항수, 정답포함 여부, **상태**:
  - `채점가능` — 정답 md 있음 → 풀이 진입 시 OMR 활성
  - `열람만` — 정답 md 없음 → PDF 열람만, "정답 미등록 — 채점 불가" 안내
  - `열람불가` — 숨김페이지수 미확정(업로드 후 추출 실패 등) → PDF 요청 시 409 "숨김 페이지수 미확정 — 정답 등록 후 열람 가능"
- **내 이력 요약**: 시험별 최근 시도 점수(있으면).
- **업로드 패널**: §5.8.

### 5.4 풀이 화면
- **레이아웃**: PDF 뷰어 + OMR 답안지 + 문항별 도구(개념/챗) — 배치는 디자인 재량(몰입 모드 권장).
- **PDF 뷰어**: **답지 제거 서브셋만 렌더**(답지는 브라우저에 아예 안 옴). 페이지 네비게이션, pdf.js 로컬 서빙.
- **OMR**: 정답 md의 문항수·과목범위 기반 그리드(과목 구분 표시). 문항마다 ①~④ 선택 + **"찍음" 체크박스**(모르고 찍었음 표시 — 채점보다 중요한 자기평가 지표). 응답은 800ms 디바운스로 **자동 임시저장** — 브라우저를 닫아도 재접속 시 이어풀기. 경과시간 라이브 표시(참고용, 제한 없음 — 자유 모드).
- **문항별 버튼**: [개념](§5.6) / [챗](§5.7).
- **제출 버튼** → 채점 결과로.
- 열람만 상태: OMR 비활성 + 안내, PDF만.

### 5.5 채점 결과 + 키워드 폼 + 이력
- **채점 결과**(제출 응답 기반): 총점 + 판정 뱃지(합격/불합격/**과락**), 과목별 점수(과락 기준 40 미달 강조), 오답 수(X), **찍었는데 맞은 수**(O+찍음), **확신 정답률**(찍음 제외 정답률 — 실력 지표), 최저 과목. 파일 3종(풀이 기록·INDEX·WRONG)이 저장소에 기록됨.
- **키워드 폼**(제출 직후, 선택): 오답·찍음 문항 목록에 "개념 키워드/메모" 인라인 입력 → 저장 시 풀이 기록과 오답 인덱스에 반영. **건너뛰기 가능**(제출은 이미 완료 상태).
- **제출 후 해제**: "원본(답지 포함) 보기" 버튼 — 제출한 시험만 원본 PDF 열람 가능.
- **시도 이력·추이**: 같은 시험 재풀이 시 시도 번호 자동 증가(1,2,3…). 시도별 점수·오답수·찍음수 추이 차트.
- **재풀이**: 제출 후 드래프트는 삭제되므로 다시 열면 빈 답안지에서 시작.

### 5.6 개념 보기 패널 (문항별)
- **데이터**: 해당 문항(`시험ID #문번`)에 `🔁 기출` 태그로 연결된:
  1. **내 개념 노트** 섹션 (내 notes/)
  2. **타인 개념 노트** 섹션 (다른 참여자 notes/, 작성자 표시, 읽기 전용)
  3. **공유 문항 해설** (_공통/풀이/{시험ID}/{문번}.md — 닉네임·날짜 서명 섹션별)
- md 렌더(제목·표·코드). 연결된 내용이 없으면 빈 상태("아직 이 문항과 연결된 정리가 없어요 — 챗으로 물어보고 정리해 보세요" + 챗 버튼).
- 노트가 CLI에서 수정되면 실시간 자동 갱신.

### 5.7 챗 패널 (문항별, agy)
- **진입**: 문항의 [챗] 버튼. **문항 컨텍스트 자동 첨부**(시험ID·문번·PDF 경로·연결 노트/해설 경로) — 첨부된 컨텍스트를 칩으로 표시.
- **대화**: 멀티턴. 답변은 스트리밍 표시. **⚠️ 표시만 될 뿐 어떤 파일에도 기록되지 않음** — 이 원칙을 UI에서 인지 가능하게.
- **"정리해줘" 승인 플로우**: 버튼 → **승인 다이얼로그**: 목적지 체크박스 [내 개념 노트]·[공유 문항 해설] (기본 둘 다) + 설명 → 승인 시 claude가 해당 md에 직접 기록(출제기준 계층에 맞는 노트 파일·섹션 자동 선정, 🔁 태그 삽입, 해설은 내 서명 섹션 append) → 완료되면 개념 보기에 실시간 반영.
- **기록 잡 상태**: 진행 표시(정리 중…). 실패/경계 위반 원복 시 경고 배너.
- CLI(agy) 미설치: 패널 진입 시 비활성 안내(503) + 설치 가이드 링크.

### 5.8 기출 업로드 + 수동 정답 폼
- **업로드**: 드롭존/파일 선택 → PDF가 명명 규칙으로 저장소에 배치 → **claude가 답지 자동 판독**(정답 md 생성 + 답지 페이지 수 판별) → 구조 검증 통과 시 목록에 "채점가능"으로 등장. 진행 단계 표시(업로드→판독→검증→등록).
- **추출 실패 시**(스캔 이미지 등): "수동 정답 입력 폼" — 과목 구성(과목명·문항 범위) 입력 → ①~④ 정답 그리드 → 저장 시 서버가 검증 후 등록. **막다른 길 없음**: 실패해도 항상 수동 경로 제공.
- claude 미설치: 자동 추출 불가 안내 + 수동 폼으로 바로 유도.
- 에티켓 고지: 저작권 자료(시중 문제집 스캔) 업로드 금지 문구.

## 7. API 요약 (화면별 가용 데이터 — 계약 변경 금지)

| 화면 | API |
|------|-----|
| 전역 | `GET /api/state`(닉네임·CLI 상태·토큰), `GET /api/events`(SSE) |
| 온보딩 | `GET /api/repo`(certs+participants 합집합), `POST /api/nickname`, `GET /api/nickname/:name/usage` `[R2]`, `DELETE /api/nickname` `[R2]` |
| 홈/자격증 | `GET /api/repo`, `POST /api/certs` `[R2]` |
| 목록 | `GET /api/exams?grade=&cert=` (grade=종류/분야) |
| 풀이 | `GET /api/exams/:id/pdf`(서브셋), `GET /api/exams/:id/omr`(구조만·정답 미포함), `GET/PUT /api/draft/:examId` |
| 제출 | `POST /api/attempts/:examId/submit` → 채점 응답, `POST /api/attempts/:examId/keywords`, `GET /api/attempts`(이력·추이), `GET /api/exams/:id/pdf-full`(제출 후) |
| 개념 | `GET /api/concept/:examId/:qno` |
| 챗 | `POST /api/chat/:examId/:qno`(스트리밍), `POST /api/chat/approve` |
| 업로드 | `POST /api/exams/upload`, `POST /api/exams/:id/answer-key`(수동 정답) |

프론트 이벤트: `qnet:attempt-submitted`(제출 응답 전달), `qnet:fs-change`(파일 변경).

## 8. 사용 시나리오 (대표 3개 — 디자인 검증용)

1. **첫 사용**: 실행 → 온보딩에서 닉네임 등록 → 자격증 등록(정보처리/정보처리기사) → 기출 PDF 업로드 → 자동 추출 완료 → 풀이 → 제출 → 채점표 확인 → 찍은 문항에 키워드 입력.
2. **일일 학습**: 실행 → (이어풀기 있으면 바로 진입) → 풀다가 모르는 문항 [챗] → 답변 마음에 들어 "정리해줘"(둘 다) → 개념 보기에서 기록 확인 → 마저 풀고 제출 → 추이에서 점수 상승 확인.
3. **복습**: 기출 재선택 → 재풀이(시도 2) → 이전 오답 문항의 [개념]에서 내/타인 정리 비교 → 제출 후 추이로 오답·찍음 감소 확인.

## 9. 제약 (디자인 시 반드시 고려)

- 무빌드 바닐라 ES 모듈 + 단일 CSS. 외부 CDN·웹폰트 금지(로컬 전용) — 시스템 폰트·유니코드/인라인 SVG 아이콘.
- 라이트/다크 모두 지원, 한국어 UI.
- 창 크기: 데스크톱 중심(PDF+OMR 병렬이 핵심), 1280px 미만 대응은 접이식.
- 서버 API·이벤트 계약 변경 불가(§7). 서버 코드 수정 불가.
