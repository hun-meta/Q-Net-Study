# Q-Net 기출 풀이 로컬 웹 앱

기출 PDF를 브라우저에서 풀고 **자동 채점·기록**하는 로컬 전용(127.0.0.1) 웹 앱입니다.
`md`/`git`이 유일한 진실 원천이며, DB는 없습니다. AI 없이도 풀이→채점→기록→열람 핵심 루프가 서버의 결정적 코드로 완결됩니다.

## 실행 전제

| 도구 | 용도 | 필수 여부 | 확인 |
|------|------|-----------|------|
| Node.js ≥ 18 | 서버 실행 | **필수** | `node -v` |
| Claude Code (`claude`) 설치 + 로그인 | 정리·기록, 정답 추출(파일 직접 쓰기) | 선택(없으면 해당 기능만 비활성) | `claude --version` |
| antigravity (`agy`) 설치 + 로그인 | 문항 챗 질의응답(표시만) | 선택(없으면 챗만 비활성) | `agy --version` |

두 CLI는 서버가 `claude --dangerously-skip-permissions` / `agy --dangerously-skip-permissions`로 호출합니다(명령은 `.qnet-web/config.json`에서 재정의 가능). 미설치·미로그인이면 상단 배너로 안내되고 해당 AI 기능만 비활성화되며, **풀이·채점·기록 열람·정답 수동 입력은 계속 동작합니다.**

## 설치 · 기동

```bash
cd web
npm install
npm start          # 127.0.0.1:4525 기본, 점유 시 4526~4535 탐색
```

브라우저에서 기동 배너에 표시된 주소로 접속 → 닉네임 선택 → 기출 목록.

## 사용 플로우

1. **닉네임 선택** — 개인 디렉토리명이 됩니다. `_공통`·경로 구분자는 쓸 수 없습니다.
2. **기출 등록/선택** — 기출 목록에서 PDF를 업로드하면 서버가 `_공통/기출문제/`에 배치하고, `claude`가 답지를 판독해 `_공통/기출문제/정답/{시험}.md`를 작성 → 구조 검증 통과 시 `INDEX.md`에 자동 부기. 검증 실패 시 **정답 수동 입력 폼**을 제공합니다(막다른 길 없음).
3. **풀이** — pdf.js 뷰어(답지 페이지는 서버가 제거한 서브셋만 전송) + OMR(①~④ / 찍음). 입력은 자동 임시저장되어 이어풀기가 됩니다.
4. **제출 · 채점** — 과목별 점수·과락·합격·확신정답률을 서버가 결정적으로 계산합니다. 제출 직후 X·찍음 문항에 **키워드/메모 폼**(건너뛰기 가능)이 뜨고 `attempt`·`WRONG.md`에 반영됩니다.
5. **추이** — 같은 시험을 다시 풀면 시도 N·점수 추이가 누적됩니다(CLI가 만든 기록도 함께 읽음).
6. **개념 보기** — 🔁 태그(`🔁 기출 {연도}-{회차}-{구분} #{문번}`, 구분 없는 구형은 필기로 해석)로 연결된 내/타인 노트와 `_공통/풀이/` 공유 해설을 함께 봅니다.
7. **챗(agy)** — 문항 컨텍스트를 담아 질문. 답변은 **표시만** 하고 파일을 쓰지 않습니다(멀티턴은 이력 재주입).
8. **승인 정리(claude)** — "이 대화 정리·기록"에서 **목적지 선택**([내 개념 노트] / [공유 문항 해설] / [둘 다], 기본 둘 다 체크) → `claude`가 저장소 규칙대로 md에 직접 기록(🔁 태그 삽입, 공유 해설은 `## {닉네임} (날짜)` 서명 섹션 append) → 워처가 화면을 즉시 갱신합니다.

## 보안 모델 (요약)

- **127.0.0.1 전용 바인딩**, `Host`/`Origin` 화이트리스트(DNS 리바인딩 차단), 상태 변경·CLI 트리거 API는 커스텀 헤더 `X-QNet-Token` 필수(드라이브바이 CSRF 차단), 허용적 CORS 헤더 미발행.
- **서버 쓰기 경계는 강제**(realpath·NFC): 타인 디렉토리·경계 밖 경로 → 403.
- **claude/agy 잡은 사후 감사**(봉쇄가 아닌 파일 무결성 탐지·복원): 잡 전 스냅샷 → 경계 위반 시 잡 전체 원자 원복, 목적지 밖 미추적 신규 삭제, 공유 해설 섹션 불변식(타인 섹션 바이트 동일), `.qnet-web/config.json`·`.git/hooks` 해시 감시, 챗 잡 무변화 검증. 위반 시 복원 + UI 경고 배너.
- 프롬프트 인젝션은 가드 시스템 프롬프트 + 사후 감사로 완화하되, `--dangerously-skip-permissions` 특성상 데이터 유출·명령 실행 자체는 사후에 막지 못함(**사용자 수용 리스크** — 신뢰 자료만 업로드).

## 트러블슈팅

| 증상 | 원인 · 해결 |
|------|-------------|
| 배너에 "agy/claude 미설치" | 해당 CLI 미설치·미로그인. `claude --version`/`agy --version` 확인 후 로그인. 챗/자동 정리·추출만 비활성이며 나머지는 정상. |
| 기동 실패 "사용 가능한 포트를 찾지 못했습니다" | 4525~4535가 모두 점유됨. 점유 프로세스 종료 후 재시도(포트는 `.qnet-web/config.json`의 `port`로 조정). |
| 챗/정리 버튼 비활성 | 상단 배너의 CLI 감지 상태 확인. 미로그인 시 `claude`/`agy`를 한 번 대화형으로 실행해 로그인. |
| 정답 추출 후 "수동 입력 폼" 표시 | 자동 구조 검증 실패(문항수 불일치·정답 도메인·숨김페이지수 범위 등) 또는 스캔 이미지. 폼으로 정답 md를 직접 작성하면 채점 가능. |
| 채점이 "열람만"으로 표시 | 해당 기출에 정답 md 미등록. 업로드로 추출하거나 정답 수동 입력. |
| UI 경고 "사후 감사" 배너 | claude/agy 잡이 승인 경계 밖을 건드려 서버가 복원함. 신뢰 자료만 업로드했는지 확인. |
| 상태 변경 요청 403 | 잘못된 `Host`/`Origin` 또는 `X-QNet-Token` 누락. 배너 주소(127.0.0.1:포트)로 접속. |

## 아키텍처 메모

- 서버: Express + chokidar 워처 + SSE. 프론트: 무빌드 바닐라 ES 모듈(pdf.js).
- CLI 브리지(`server/cliBridge.js`)는 `execFile`/`spawn` 인자 배열만 사용(셸 보간 없음), 잡 큐 동시 1, Node 타이머 kill 타임아웃(추출 5분/정리 2분/챗 3분). macOS에 `timeout` 명령이 없어 쉘 timeout에 의존하지 않습니다.
- 규칙 문서: 저장소 루트 [`CLAUDE.md`](../CLAUDE.md), 스킬 `/exam-pdf`·`/study-note`·`/attempt-log`.
