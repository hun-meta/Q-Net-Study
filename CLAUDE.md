# Q-Net Study — 국가자격증 공유 스터디 저장소

Q-Net(큐넷, 한국산업인력공단)에서 신청 가능한 자격증을 여러 명이 함께 공부하고 기록하는 저장소입니다.
Claude는 이 문서의 규칙을 모든 작업에서 따릅니다. 모든 산출물(노트, 기록, 커밋 메시지)은 한국어로 작성합니다.

## 디렉토리 구조

```
{종류}/{자격증}/{닉네임}/{md 파일들}
```

- **종류**: **분야(직무·주제) 단위 디렉토리** — 예: `정보처리/`, `전기/`, `안전관리/`. 같은 분야의 여러 등급(기사·산업기사 등)을 한 폴더로 묶는다.
  - 디렉토리명은 자유롭게 정할 수 있으며 화이트리스트가 없다. 단 `web`·`docs`·`templates`·`node_modules` 및 `.`/`_`로 시작하는 이름은 예약(시스템·공용)이라 종류로 쓸 수 없다.
  - 자격 **등급**(기사/산업기사/기능사 등)은 디렉토리 그룹이 아니라 각 자격증의 `_공통/info.md` 메타데이터(개요 표의 "등급" 행)로 기록한다.
- **자격증**: 정식 종목명 그대로 (예: `정보처리기사`, `산업안전기사`, `전기기사`)
- **닉네임**: 개인 공간. 다른 사람의 닉네임 디렉토리는 **읽기만 하고 절대 수정하지 않는다.**

자격증 디렉토리 내부 표준 레이아웃:

```
정보처리/정보처리기사/
├── _공통/                      # 참여자 전원이 공유 (누구나 수정 가능)
│   ├── info.md                 # 자격증 개요: 시험 구조, 과목, 합격 기준, 일정
│   ├── 출제기준/                # 과목별 출제기준 (주요항목 → 세부항목 계층)
│   │   └── {과목명}.md
│   ├── 기출문제/                # 기출 PDF 저장소 (명명 규칙 아래 참조)
│   │   ├── INDEX.md            # 기출 PDF 목록 인덱스
│   │   ├── 2023-1-필기.pdf
│   │   └── 정답/               # 문항별 정답 md (수동 작성 또는 웹 앱이 추출)
│   │       └── {연도}-{회차}-{구분}.md
│   └── 풀이/                    # 문항별 공유 해설 (닉네임 서명 섹션 append-only)
│       └── {연도}-{회차}-{구분}/
│           └── {문번}.md
└── hun/                        # 개인 공간 (닉네임)
    ├── README.md               # 개인 학습 현황판 (목표 시험일, 진행률)
    ├── notes/                  # 개념 정리
    │   └── {과목명}/
    │       └── {NN}-{주요항목명}.md
    └── attempts/               # 기출 풀이 기록
        ├── INDEX.md            # 풀이 이력 인덱스 (자동 갱신)
        ├── WRONG.md            # 오답·불확신 문항 인덱스 (자동 갱신)
        └── {연도}-{회차}-{구분}-{시도N}.md
```

## 파일 명명 규칙

| 대상 | 규칙 | 예시 |
|------|------|------|
| 기출 PDF | `{연도}-{회차}-{구분}.pdf` | `2023-1-필기.pdf`, `2022-3-실기.pdf` |
| 풀이 기록 | `{연도}-{회차}-{구분}-{시도N}.md` | `2023-1-필기-1.md` (1차 시도) |
| 개념 노트 | `{NN}-{출제기준 주요항목명}.md` (NN = 출제기준 순번 2자리) | `01-요구사항확인.md` |
| 출제기준 | `{과목명}.md` | `소프트웨어설계.md` |

- 구분 값은 `필기` 또는 `실기`만 사용한다.
- 회차는 숫자만 쓴다 (`1회` ❌ → `1` ⭕). CBT 상시 시험은 응시일로 구분: `2024-0415상시-필기.pdf`.

## 개념 정리 원칙

- 노트는 반드시 **출제기준의 계층(과목 → 주요항목 → 세부항목 → 세세항목)을 따라** 작성한다. 임의 목차를 만들지 않는다.
  - 매핑: 주요항목 = 노트 파일 1개, 세부항목 = `##` 섹션, 세세항목 = 섹션 내 키워드.
  - 출제기준은 통상 3년 주기로 개편되므로 적용 기간을 `_공통/info.md`에 기록한다.
- 노트 작성 전 `_공통/출제기준/{과목명}.md`가 있는지 확인하고, 없으면 먼저 조사해 만든다.
- 템플릿: `templates/note.md`
- 기출에서 반복 출제된 개념은 노트에 `🔁 기출 {연도}-{회차}-{구분} #{문번}` 형태로 역참조를 남긴다 (구분 = `필기`/`실기`).
  - 하위호환: 구분을 생략한 구형 표기 `🔁 기출 {연도}-{회차} #{문번}`도 인식되며 **필기**로 해석한다. 새 노트는 구분 포함형으로 쓴다.

## 기출 풀이 기록 원칙

- 풀이 기록은 `templates/attempt.md` 양식을 따른다. 문항마다 **내 답 / 정답 / 결과(O·X) / 확신도**를 기록한다.
- 확신도는 3단계 고정값: `확신`(근거를 알고 선택) / `애매`(둘 중 고민) / `찍음`(모르고 선택).
- 복습 우선순위: `X+찍음` > `X+애매` > `X+확신`(개념 오해, 가장 위험) > `O+찍음`(운으로 맞음) > `O+애매`.
- 풀이 기록을 만들거나 수정하면 **반드시 `attempts/INDEX.md`와 `attempts/WRONG.md`를 함께 갱신**한다 (`/attempt-log` 스킬이 담당).

## 스킬 (Claude Code 프로젝트 스킬)

| 스킬 | 용도 |
|------|------|
| `/cert-init` | 새 자격증 공간 또는 개인 공간 생성 (조사 → `_공통/` 구축 → 개인 디렉토리) |
| `/study-note` | 출제기준 섹션 기반 개념 정리 노트 작성·보강 |
| `/exam-pdf` | 기출문제 PDF 등록·판독: 명명 규칙 적용, 문항 추출, `기출문제/INDEX.md` 갱신 |
| `/attempt-log` | 기출 풀이 기록 작성 + `INDEX.md`/`WRONG.md` 자동 인덱싱 |
| `/review` | 오답·불확신 문항 복습 세션 (WRONG.md 기반 재출제·재채점) |

## 공유 문항 해설 (`_공통/풀이/`)

- 문항별 해설은 `_공통/기출문제/정답/{연도}-{회차}-{구분}.md`(정답)와 별개로 `_공통/풀이/{연도}-{회차}-{구분}/{문번}.md`에 모은다.
- 한 파일에 여러 사람이 기여할 수 있으며, **각자 자기 서명 섹션에만 append** 한다:
  ```markdown
  ## {닉네임} ({YYYY-MM-DD})
  (내 해설 내용)
  ```
- **다른 사람의 `## {닉네임}` 섹션은 절대 수정·삭제하지 않는다** (append-only). 파일·디렉토리가 없으면 최초 생성해도 된다.
- 정답 md 포맷: frontmatter `문항수·숨김페이지수·추출도구·추출일` + 과목 섹션 `## {과목명} ({시작}-{끝})` + `| 문번 | 정답 |` 표. 수동 작성도 동일 포맷을 따른다.

## 공용 저장소 에티켓

- `_공통/`은 모두가 수정 가능. 개인 디렉토리는 본인만 수정한다.
- 커밋 메시지 형식: `[{자격증}/{닉네임}] 요약` (공통 자료는 `[{자격증}/공통] 요약`).
- 대용량 PDF는 100MB를 넘지 않게 하고, 저작권상 재배포가 문제되는 자료는 링크로만 남긴다.
- Claude는 사용자가 명시하지 않는 한 커밋하지 않는다.

## 참고: Q-Net 자격 체계 요약

- 국가기술자격 등급: **기술사 > 기능장 > 기사 > 산업기사 > 기능사** (기능사는 응시 제한 없음, 상위 등급은 학력·경력 요건 있음)
- 필기 합격 기준(기사 기준): 과목당 40점 이상 & 전 과목 평균 60점 이상. 실기: 60점 이상.
- 필기 합격 후 2년간 실기 응시 가능.
- 상세 조사 자료는 `docs/qnet-guide.md` 참조.
