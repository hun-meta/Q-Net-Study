# study-note — 출제기준 기반 개념 정리

## 위치 규칙

- 노트 경로: `{종류}/{자격증}/{닉네임}/notes/{과목명}/{NN}-{주요항목명}.md`
- `NN` = `_공통/출제기준/{과목명}.md`에 적힌 주요항목 순번 (2자리).
- 닉네임을 모르면 사용자에게 확인한다. **본인 디렉토리에만 쓴다.**

## 실행 절차

### 1단계: 출제기준 확인
- `_공통/출제기준/{과목명}.md`를 읽는다.
- **없으면** 먼저 조사해서 만든다 (`/cert-init` 2단계와 동일한 방식). 출제기준 없이 임의 목차로 노트를 만들지 않는다.

### 2단계: 노트 작성
- `templates/note.md` 양식을 따른다.
- 구조 매핑: **주요항목 = 파일 1개**, **세부항목 = `##` 섹션**, **세세항목 = 섹션 안의 키워드/소제목**.
- 내용 작성 원칙:
  - 정의 → 왜 중요한가 → 상세 → 암기 포인트 순으로, 시험에 나오는 수준까지만 깊이 있게.
  - 비교되는 개념은 표로 정리한다 (시험은 차이점을 묻는다).
  - 계산 문제가 나오는 항목은 공식 + 예제 1개를 반드시 포함한다.
  - 웹 조사로 보강하되, 출처가 갈리는 내용은 표시해 둔다.

### 3단계: 기출 연계 (기출 데이터가 있을 때)
- `_공통/기출문제/`에 해당 과목 기출이 등록되어 있으면, 이 주요항목에서 출제된 문항을 찾아 `🔁 기출 {연도}-{회차}-{구분} #{문번}` 형태(구분 = 필기/실기)로 노트에 역참조를 남긴다. 구분 없는 구형 표기는 필기로 해석된다(하위호환).
- 사용자의 `attempts/WRONG.md`에 이 항목 관련 오답이 있으면 해당 개념을 **중점 보강**하고 ⭐ 표시한다.

### 4단계: 진행률 갱신
- 개인 `README.md`의 과목별 진행률 체크리스트에서 이 주요항목을 체크한다.
- frontmatter의 `진행도`(정리중/1회독/복습완료)와 `갱신일`을 갱신한다.

## 규칙
- 한 번에 한 주요항목씩. "과목 전체 정리" 요청이면 주요항목 목록을 보여주고 순서대로 진행한다.
- 이미 노트가 있으면 덮어쓰지 말고 **보강**한다 (기존 내용 유지 + 추가).
