# cert-init — 자격증 스터디 공간 생성

## 입력 확인

1. **자격증명**과 **닉네임**을 파악한다. 없으면 물어본다.
2. 자격증의 **종류(분야)** 를 정한다: 그 자격증의 직무·주제 분야명이 최상위 디렉토리가 된다 (예: 정보처리기사·정보처리산업기사 → `정보처리/`, 전기기사 → `전기/`, 산업안전기사 → `안전관리/`). 같은 분야의 다른 등급 자격증이 이미 있으면 **그 분야 디렉토리를 재사용**한다. 분야명은 자유이되 `web`·`docs`·`templates`·`node_modules` 및 `.`/`_`로 시작하는 이름은 예약이라 쓰지 않는다. 애매하면 사용자에게 분야명을 확인한다.
   - 자격 **등급**(기사/산업기사/기능사/기능장/기술사, 개별법 자격 등)은 디렉토리 그룹이 아니라 `_공통/info.md`의 개요 표 "등급" 행에 메타데이터로 기록한다.

## 실행 절차

### 1단계: 기존 공간 확인
- `{종류}/{자격증}/`이 이미 있으면 → `_공통/` 구축은 건너뛰고 **개인 공간만** 추가한다 (3단계로).
- `{종류}/{자격증}/{닉네임}/`도 이미 있으면 → 현황만 보고하고 종료한다.

### 2단계: 자격증 조사 및 `_공통/` 구축
WebSearch/WebFetch로 조사한다 (Q-Net 페이지는 JS 렌더링이라 직접 fetch가 안 될 수 있음 → 검색 결과 교차 검증):
- 시험 구조 (필기/실기 형태, 과목, 문항 수, 시간)
- 합격 기준, 올해 시험 일정, 최근 합격률
- **최신 출제기준** (적용 기간 확인 필수): 과목 → 주요항목 → 세부항목 → 세세항목

생성 파일:
- `_공통/info.md` — `templates/cert-info.md` 양식으로 작성 (개요 표에 자격 **등급** 행을 포함해 등급 메타데이터를 남긴다)
- `_공통/출제기준/{과목명}.md` — 과목별 1파일. 주요항목에 `01`, `02`… 순번을 붙여 계층 그대로 기록 (이 순번이 노트 파일명의 NN이 된다)
- `_공통/기출문제/INDEX.md` — 빈 인덱스 (표 헤더만: 파일명 | 연도 | 회차 | 구분 | 문항수 | 정답포함 | 등록자 | 비고)

### 3단계: 개인 공간 구축
```
{종류}/{자격증}/{닉네임}/
├── README.md          # 목표 시험일, 과목별 진행률 체크리스트 (출제기준 기반)
├── notes/{과목명}/     # 과목별 빈 디렉토리 (.gitkeep)
└── attempts/
    ├── INDEX.md       # 표 헤더만: 시험 | 시도 | 날짜 | 총점 | 과목별 | 결과 | 오답수 | 찍음수
    └── WRONG.md       # 헤더만 (## 과목별 섹션은 기록 생기면 추가)
```

### 4단계: 보고
생성한 구조를 트리로 보여주고, 다음 단계(`/study-note`로 개념 정리 시작, `/exam-pdf`로 기출 등록)를 안내한다.

## 규칙
- 다른 사람의 닉네임 디렉토리는 절대 건드리지 않는다.
- 출제기준을 확정 못 한 경우 임의로 지어내지 말고, 찾은 범위까지만 기록하고 "원문 확인 필요"를 명시한다.
- 커밋은 사용자가 요청할 때만 한다.
