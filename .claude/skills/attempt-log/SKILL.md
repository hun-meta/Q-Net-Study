# attempt-log — 기출 풀이 기록 + 인덱싱

## 입력 확인

- 대상 시험(`{연도}-{회차}-{구분}`)과 닉네임을 파악한다.
- 시도 번호: `attempts/`에 같은 시험 기록이 있으면 마지막 시도 +1, 없으면 1.
- 파일 경로: `{종류}/{자격증}/{닉네임}/attempts/{연도}-{회차}-{구분}-{시도N}.md`

## 실행 절차

### 1단계: 답 수집
사용자에게 받는 형식은 자유 (예: "1번 3, 2번 1(찍음), 3번 4…" / 표 붙여넣기 / 과목별로 나눠서).
문항마다 두 가지를 받는다:
- **내 답** (①②③④ 또는 1~4)
- **확신도**: `확신` / `애매` / `찍음` — 말 안 한 문항은 `확신`으로 간주하되, 시작할 때 이 규칙을 안내한다.

### 2단계: 채점
- 정답 출처 우선순위: ① `_공통/기출문제/` PDF의 정답표 (Read로 추출) ② 사용자 제공 정답표.
- PDF에 정답이 없고 사용자도 모르면 웹에서 찾되, **출처가 불확실한 정답은 `?` 표시**하고 확정하지 않는다.
- 과목별 점수 계산: `(과목 정답 수 / 과목 문항 수) × 100`. 과목당 40점 미만 = 과락, 평균 60 이상 = 합격.

### 3단계: 풀이 기록 작성
- `templates/attempt.md` 양식으로 작성한다.
- X / 찍음 / 애매 문항은 **개념 키워드**를 반드시 채운다 (PDF에서 해당 문항을 읽고 무슨 개념인지 판별. 가능하면 출제기준 주요항목에 매핑).
- "오답·불확신 정리" 섹션에 문항별 오개념·올바른 근거를 정리한다 (사용자와 대화하며 채우거나, PDF 해설 기반).

### 4단계: 인덱싱 (필수 — 건너뛰지 않는다)

**`attempts/INDEX.md`** — 풀이 이력 표에 행 추가:

| 시험 | 시도 | 날짜 | 총점 | 과목별 최저 | 결과 | X | O+찍음 | 확신정답률 | 기록 |
|------|------|------|------|------------|------|---|--------|-----------|------|
| 2023-1-필기 | 1 | 2026-07-20 | 62.0 | 45 (DB구축) | 합격권 | 18 | 7 | 84% | [링크](2023-1-필기-1.md) |

표 아래 "추이" 섹션에 같은 시험 재시도 시 점수 변화를 남긴다.

**`attempts/WRONG.md`** — 오답·불확신 인덱스. 대상: `X 전부` + `O+찍음` + `O+애매`.
과목별로 그룹핑하고, 복습 우선순위 순서로 정렬한다:

```markdown
# 오답·불확신 인덱스

> 우선순위: 🔴 X+찍음 > 🟠 X+애매 > ⛔ X+확신(오개념!) > 🟡 O+찍음 > ⚪ O+애매
> 복습(재풀이)에서 확신+정답 2회 연속이면 취소선 처리로 졸업.

## {과목명}

- [ ] 🔴 2023-1-필기 #23 {개념 키워드} → [기록](2023-1-필기-1.md) / [노트](../notes/{과목}/{NN}-{항목}.md) (복습 0회)
- [ ] ⛔ 2023-1-필기 #31 {개념 키워드} — 확신하고 틀림, 개념 재점검 필요
```

같은 개념이 여러 시험에서 반복되면 한 항목으로 합치고 출처 시험을 나열한다 (빈출 표시 `🔁×n`).
노트에 남기는 기출 역참조 태그는 CLAUDE.md 규칙대로 구분 포함형 `🔁 기출 {연도}-{회차}-{구분} #{문번}`을 쓴다 (구분 없는 구형 표기는 필기로 해석 — 하위호환).

### 5단계: 보고
총점·과목별 점수·합격 여부, 확신 정답률, 가장 약한 과목/주요항목, 추천 다음 행동(`/review` 또는 `/study-note` 보강)을 요약한다.

## 규칙
- 인덱스 갱신 없이 풀이 기록만 만들고 끝내지 않는다 (INDEX.md + WRONG.md는 항상 동기화).
- 다른 사람의 attempts에는 절대 쓰지 않는다.
- 점수 계산을 어림하지 않는다 — 문항 수를 세어 정확히 계산한다.
