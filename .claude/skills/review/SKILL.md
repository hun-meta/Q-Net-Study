# review — 오답 복습 세션

## 준비

1. 닉네임 확인 후 `{종류}/{자격증}/{닉네임}/attempts/WRONG.md`를 읽는다.
2. 복습 대상 선정 (기본 10문항, 사용자가 과목·개수를 지정하면 그에 따름):
   - 우선순위: 🔴 X+찍음 > 🟠 X+애매 > ⛔ X+확신 > 🟡 O+찍음 > ⚪ O+애매
   - 같은 우선순위면 복습 횟수가 적은 것부터.

## 세션 진행

### 문항별 루프
1. `_공통/기출문제/` PDF에서 해당 문항 원문을 추출해 **보기 순서까지 그대로** 다시 출제한다.
   - PDF에서 추출 불가하면 풀이 기록의 개념 키워드로 개념 질문을 만들어 대체한다.
2. 사용자의 답 + 확신도를 받는다.
3. 즉시 채점하고 피드백:
   - 정답이어도 **왜 그게 답인지** 근거를 한 줄로 설명해 준다.
   - 오답이면 오개념을 짚고, 연결된 노트(`notes/`)의 해당 섹션을 안내한다. 노트에 그 개념이 없으면 `/study-note` 보강을 제안한다.

### 세션 종료 시: WRONG.md 상태 갱신
- 문항별 복습 횟수와 최근 결과를 갱신: `(복습 2회, 최근 ○확신)`
- **졸업 조건**: 확신+정답 **2회 연속** → 체크박스 체크 + 취소선 처리 (삭제하지 않고 이력 보존).
- ⛔(확신하고 틀렸던) 문항은 졸업 조건을 3회 연속으로 강화한다.
- 여전히 틀리는 문항은 우선순위를 유지하고 `🔥` 표시를 붙인다.

## 세션 요약 보고
- 오늘 푼 수 / 맞춘 수 / 졸업한 문항 수
- 남은 오답 수 (우선순위별)
- 반복해서 틀리는 개념 → `/study-note`로 보강할 주요항목 추천

## 규칙
- 답을 먼저 보여주지 않는다 (한 문항씩 출제 → 답 받기 → 채점).
- WRONG.md 갱신을 빼먹지 않는다 — 세션의 산출물은 갱신된 인덱스다.
