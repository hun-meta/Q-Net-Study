# exam-pdf — 기출문제 PDF 등록·판독

## 모드 판별

- 사용자가 **새 PDF 파일**을 주면 → [등록 모드]
- 이미 등록된 PDF에 대한 질문/분석 요청이면 → [판독 모드]

## 등록 모드

### 1단계: PDF 내용 확인
- Read 도구로 PDF를 읽는다 (10페이지 초과 시 `pages`로 나눠 읽는다. 먼저 1~5페이지로 표지·구성 파악).
- 파악할 것: 자격증명, 연도, 회차, 필기/실기 구분, 문항 수, **정답/해설 포함 여부**.
- 파일 내용과 사용자가 말한 정보가 다르면 지적하고 확인받는다.

### 2단계: 명명 규칙 적용 및 배치
- 규칙: `{연도}-{회차}-{구분}.pdf` (예: `2023-1-필기.pdf`, CBT 상시는 `2024-0415상시-필기.pdf`)
- 대상 위치: `{종류}/{자격증}/_공통/기출문제/`
- 원본을 `cp`로 복사해 배치한다 (사용자 원본은 삭제하지 않는다). 동명 파일이 있으면 덮어쓰기 전 확인받는다.

### 3단계: INDEX.md 갱신
`_공통/기출문제/INDEX.md`에 행 추가 (연도 내림차순 정렬 유지):

| 파일명 | 연도 | 회차 | 구분 | 문항수 | 정답포함 | 등록자 | 비고 |
|--------|------|------|------|--------|----------|--------|------|
| [2023-1-필기.pdf](2023-1-필기.pdf) | 2023 | 1 | 필기 | 100 | O | hun | 해설 포함 |

### 4단계: 구조 요약 보고
- 과목별 문항 범위 (예: 1~20 소프트웨어 설계), 정답표 위치(페이지), 특이사항을 보고한다.
- 풀 준비가 되면 `/attempt-log`로 풀이 기록을 시작하라고 안내한다.

## 판독 모드

기출 PDF에 대한 조사·질문 처리:
- **문항 추출**: 특정 문항/과목의 문제와 보기를 텍스트로 추출 (풀이 기록 채점, 오답 정리에 사용)
- **정답표 추출**: 정답 페이지를 읽어 `문번 → 정답` 표로 정리 (`/attempt-log` 채점에 사용)
- **출제 경향 분석**: 문항들을 출제기준(`_공통/출제기준/`)의 주요항목에 매핑해 어느 항목에서 몇 문제 나왔는지 집계
- **개념 조사**: 특정 문항의 개념을 설명하고, 관련 노트(`notes/`)와 연결

## 웹 앱 연동 (상호 참조)

- 로컬 웹 앱(`web/`)에서도 기출 PDF를 등록할 수 있다: 기출 목록 화면의 업로드 드롭존 → `POST /api/exams/upload`가 PDF를 `_공통/기출문제/`에 배치하고, `claude`가 답지를 판독해 `_공통/기출문제/정답/{연도}-{회차}-{구분}.md`(정답 md)를 작성한 뒤 `INDEX.md`를 갱신한다.
- 정답 md 포맷: frontmatter `문항수·숨김페이지수·추출도구·추출일` + 과목 섹션 `## {과목명} ({시작}-{끝})` + `| 문번 | 정답 |` 표. 이 스킬로 수동 판독할 때도 같은 포맷으로 저장하면 웹 채점과 호환된다.
- `INDEX.md`는 `숨김페이지수` 칼럼을 포함한 9칼럼 형식(구형 8칼럼은 자동 하위호환, 부재 시 기본 1)으로 관리된다. 웹 앱은 이 값으로 풀이 화면에 답지 페이지를 숨겨 전송한다.
- 자세한 사용법·트러블슈팅은 `web/README.md` 참조.

## 규칙
- PDF가 스캔 이미지라 텍스트 추출이 안 되면, 읽을 수 있는 범위(이미지 판독)로 처리하고 한계를 명시한다.
- 저작권 주의: 시중 문제집 스캔본은 등록하지 말도록 안내한다 (Q-Net 공개 기출, 자작 복원 문제는 OK).
- 문항 내용을 추출할 때 정답을 임의로 추정하지 않는다 — 정답표가 없으면 "정답 미포함"으로 기록한다.
