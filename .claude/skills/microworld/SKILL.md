# microworld — 코드를 조작하며 이해하는 일회용 마이크로월드

> 원안: Geoffrey Litt "Understanding is the new bottleneck"(2026)의 마이크로월드/AI HUD 기법 + Seymour Papert "Mathland".
> 대상은 **이 저장소의 코드**다(자격증 시험 개념이 아니다 — 그건 웹 앱의 마이크로월드 생성 기능이 담당한다).

## 왜 이 스킬이 있는가

에이전트가 코드를 대량으로 짜면 사람이 "무슨 일이 벌어지는지" 감을 잃기 쉽다(인지 부채). 로그를 줄줄이 읽는 대신, **코드의 내부 상태를 눈에 보이게 만들고 값을 바꿔가며 돌려보면** 동작 방식이 손에 잡힌다. Papert의 "Mathland"처럼, 이해는 설명을 읽는 것보다 **직접 조작**에서 온다.

마이크로월드는 **출시할 소프트웨어가 아니라 나를 위한 일회용 시뮬레이션**이다. 릿의 표현으로는 "AI HUD" — 대화형 코파일럿이 아니라, 코드가 어떻게 도는지에 대한 *새로운 감각*을 주는 계기판이다. 버그를 고치더라도, 마이크로월드 안에서 고치면 그 과정에서 기계에 대한 감을 얻는다. 에이전트에게 버그 수정만 맡기면 그 주변 통찰은 사라진다.

## 언제 쓰나

- 에이전트가 방금 짠/고친 코드의 동작이 머릿속에 안 그려질 때.
- `git diff`나 `explain-diff` 문서만으로는 "실행 시 상태가 어떻게 전개되는지"가 안 잡힐 때.
- 알고리즘·상태기계·파이프라인처럼 **입력 → 단계별 내부 상태 → 출력**이 있는 코드.

`explain-diff`(정적 설명 문서)와 짝이다: explain-diff는 변경을 **읽게** 하고, microworld는 동작을 **조작하게** 한다.

## 이 저장소에서 좋은 대상

- **채점 파이프라인** (`web/server/grading.js`, `passCriteria.js`): 모의 답안 → 과목별 점수 → 과락/평균/합격 판정이 단계별로 계산되는 과정.
- **사후 감사 복원** (`web/server/audit.js`): 스냅샷 → CLI 잡 실행 → 경계 위반 탐지 → 원자적 원복. 어떤 변경이 허용/원복되는지 시각화.
- **태그 인덱스 스캔** (`web/server/tagIndex.js`): `🔁 기출` 태그가 파싱되어 `byQuestion` 역인덱스로 구성되는 과정, mtime 증분 캐시.
- **PDF 서브셋** (`web/server/pdfSubset.js`): 답지 페이지가 제거된 서브셋이 만들어지는 과정.
- **SSE·워처** (`web/server/sse.js`, `watcher.js`): 파일 변경 → 브로드캐스트 → 프론트 반영 흐름.
- **프론트 라우팅/스토어** (`web/public/js/router.js`, `store.js`): 상태 전이.

## 설계 원칙 (릿의 실천 패턴)

1. **단일 파일·무빌드**: CSS·JS 인라인, 외부 CDN/의존성 0. `file://`로 열어도 완전히 동작(이 저장소 프론트와 같은 무빌드 철학).
2. **실제 코드 로직을 반영**: 대상 소스를 읽고, 그 알고리즘/상태 전이를 마이크로월드 JS에 충실히 옮긴다. 가능하면 **결정적 로직은 Node로 먼저 검증**한 뒤 HTML에 넣는다(이 스킬로 만든 채점·감사 등은 서버 코드와 결과가 일치해야 의미가 있다).
3. **구체 예시 하나를 내장**: 빈 화면 금지. 의미 있는 기본 입력(예: 프로세스 4개, 모의 답안 1세트)을 채워 열자마자 돌아가게.
4. **직관 우선**: 시뮬레이션 위에 "이 코드가 무엇을 하는가" 1~2문장. 세부는 조작하면 드러나게.
5. **조작·스크럽 가능**: 입력을 바꾸거나(슬라이더·폼), **타임라인을 단계별로 스크럽**해 각 단계의 내부 상태를 훑게 한다(릿의 핵심 패턴). 단계에 주석을 달 수 있으면 더 좋다.
6. **빠른 초안 → 대화로 다듬기**: 완벽을 노리지 말고 동작 초안부터. 사용자와 "이 상태도 보여줘"로 키우고, 안 되는 기능은 버린다. **생성된 UI 코드 자체는 검증하지 않는다** — "동작/문제가 보이는가"로만 판단한다.

## 출력

- 저장 위치: **`.omc/microworld/YYYY-MM-DD-<슬러그>.html`** (gitignore되는 재생성 가능 영역, 일회성). 디렉토리가 없으면 만든다.
- 라이트/다크 대응, 반응형. 다이어그램은 ASCII 금지 — HTML/SVG/Canvas로.
- 실행 후 절대경로와 여는 법(`open <경로>`), 조작법 2~3줄을 안내한다.

## 규칙

- 대상 소스는 **읽기만** 한다 — 이 스킬은 이해용 시각화만 만들고 원본 코드를 고치지 않는다.
- 산출물은 `.omc/`(gitignore) 밖에 커밋하지 않는다 — 개인용 일회성 학습 자료다.
- 커밋은 사용자가 요청할 때만.
- 시험 개념(스케줄링·정규화 등) 학습용 시뮬레이션이 필요하면 이 스킬이 아니라 **웹 앱의 마이크로월드 생성 기능**(개념 보기 → 생성)을 쓴다.
