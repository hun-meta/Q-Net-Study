# 2025-1회-필기 2번

> 블랙박스 테스트에 대한 설명으로 옳지 않은 것은? — 정답 ④ (기초 경로 검사·제어 구조 검사는 화이트박스 기법)

## 준성 (2026-07-22)

**정답 근거**: 블랙박스 테스트는 소프트웨어의 내부 구조나 작동 원리를 보지 않고, 겉으로 드러나는 기능과 인터페이스(입력과 출력)에만 초점을 맞춰 검사하는 방식이다. 내부를 알 수 없는 '검은 상자'처럼 취급해, 특정 값을 입력했을 때 기대하는 결과가 정확히 나오는지 확인한다. ④의 **기초 경로 검사·제어 구조 검사는 내부 구조를 직접 확인하는 화이트박스 테스트의 종류**이므로 옳지 않은 설명 → 정답.

- ① 옳음: 각 기능이 완전히 작동되는 것을 입증하는 테스트 (기능 관점).
- ② 옳음: 프로그램의 내부 구조를 고려하지 않는다.
- ③ 옳음: 소프트웨어 인터페이스에서 실시된다.
- ④ 틀림: 기초 경로 검사·제어 구조 검사는 **화이트박스** 기법.

**함정 포인트**: 기법 이름에 "경로·구조"처럼 코드 내부를 들여다보는 단어가 있으면 화이트박스로 분류하면 된다.
