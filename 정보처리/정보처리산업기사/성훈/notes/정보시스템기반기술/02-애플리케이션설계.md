# 02. 애플리케이션설계

> 출처: `_공통/출제기준/정보시스템기반기술.md`의 주요항목 순번을 따른다.

## 공통 모듈 설계

### 핵심 개념
- **소프트웨어 아키텍처**: 시스템 구조를 정의하는 설계 활동. 4+1 뷰(논리·구현·프로세스·배포 뷰 + 유스케이스), 아키텍처 패턴(파이프 필터, 클라이언트-서버, SOA, MVC 등).
- **모듈 설계**: 결합도(Coupling, 낮을수록 좋음)와 응집도(Cohesion, 높을수록 좋음)로 모듈 품질 평가. 모듈화 = 기능 분리로 복잡성 감소.
- **설계 모델링**: UML(구조·행위 다이어그램), DFD(자료 흐름도: 프로세스·데이터 흐름·데이터 저장소·단말), HIPO(도식 목차·총괄 도표·세부 도표).
- **데이터 모델링**: E-R 모델(개체=사각형, 속성=타원, 관계=마름모), 관계 대수(순수 연산: Select·Project·Join·Division).

### 암기 포인트
- ⭐ 결합도 강한 순서: **내용 > 공통 > 외부 > 제어 > 스탬프 > 자료** (자료 결합도가 가장 약=좋음).
- ⭐ 아키텍처 4+1 뷰 = **논리·구현(컴포넌트)·프로세스·배포** + 유스케이스. "구현 뷰"≠"물리적 뷰".
- ⭐ **SOA** = 서비스 단위·컴포넌트 중심으로 공유·재사용 가능하게 구축하는 IT 아키텍처.
- ⭐ E-R: **개체=사각형(□)**, **속성=타원(○)**, **관계=마름모(◇)**. 관계 대수 순수 연산 8가지 아닌 것 → 일반 집합 연산(Union·Intersection·Difference·Cartesian).

## 객체지향 설계

### 핵심 개념
- **객체지향(OOP)**: 객체(데이터+메소드) 중심 프로그래밍. 핵심 개념 = 캡슐화(정보 은닉), 상속(재사용), 다형성, 추상화. 클래스 = 유사한 객체의 공통 특성을 묶은 일반적 타입.
- **주요 용어**: 메소드(객체가 메시지를 받아 실행할 구체적 연산), 추상 클래스(인스턴스 생성 불가, 공통점만 모은 클래스, 하위가 상속받아 구체화).
- **디자인 패턴(GoF)**: 생성(Creational)·구조(Structural)·행위(Behavioral) 3종류. Singleton(인스턴스 하나 보장), Observer(상태 변화 전파), Decorator(동적 기능 추가), Adapter(인터페이스 변환).

### 암기 포인트
- ⭐ **캡슐화** = 데이터+메소드를 하나로 묶음 + 정보 은닉(외부 직접 접근 차단).
- ⭐ **추상 클래스** = 인스턴스 생성 **불가**, 공통 기능만 모음 → 하위 클래스가 상속받아 구체화.
- ⭐ 디자인 패턴 분류: **생성**(Singleton·Factory·Builder) / **구조**(Decorator·Adapter·Proxy) / **행위**(Observer·Strategy·Iterator).
- ⭐ **Observer** = "한 객체 상태 변화 → 의존 객체들에 자동 통지" / **Adapter** = "기존 클래스 재사용 위해 중간에서 맞춰주는 역할".
