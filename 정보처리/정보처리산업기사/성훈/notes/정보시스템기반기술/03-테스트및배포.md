> 출처: `_공통/출제기준/정보시스템기반기술.md`의 주요항목 순번을 따른다.

## 애플리케이션 테스트 수행

### 핵심 개념
- **테스트 단계**: 단위 테스트(개별 모듈) → 통합 테스트(모듈 간 인터페이스) → 시스템 테스트 → 인수 테스트(알파: 개발자 환경, 베타: 사용자 환경).
- **테스트 기법**: 블랙박스(명세 기반, 입출력 중심 — 동등 분할·경계값 분석·루프 검사), 화이트박스(내부 로직 기반 — 구문·분기·조건 검사).
- **루프 검사(Loop Test)**: 단순루프·중첩루프·연쇄루프·비정규루프 4가지 반복 구조 검사.
- **디버깅**: 오류가 발생한 코드를 추적하여 수정하는 작업.

### 암기 포인트
- ⭐ **단위 테스트** = 개별 모듈이 정확히 구현되었는지, 예정 기능이 제대로 수행되는지 점검.
- ⭐ **통합 테스트** = 모듈 간 인터페이스·결합 검사 (빅뱅·상향식·하향식).
- ⭐ **알파 테스트** = 통제된 환경(개발자 앞)에서 사용자가 수행, 오류를 함께 확인·기록.
- ⭐ 루프 검사 4종류 = **단순·중첩·연쇄·비정규** 루프.

## UI 테스트

### 핵심 개념
- **UI(User Interface)**: 사용자가 시스템을 쉽게 조작할 수 있도록 만드는 접점. 기본 원칙 = 직관성·유효성·학습성·유연성·유용성.
- **GUI(Graphic User Interface)**: 아이콘·메뉴를 마우스로 선택하여 작업을 수행하는 그래픽 기반 UI.
- **UI/UX**: UI = 사용자 인터페이스 자체, UX = 사용자가 경험하는 전반적 만족도. UI 테스트 = 사용성·접근성·디자인 일관성 검증.

### 암기 포인트
- ⭐ UI 기본 원칙: **직관성**(누구나 쉽게) · **유효성**(정확한 목적 달성) · **학습성**(처음도 쉽게) · **유연성**(다양한 상황 대응).
- ⭐ GUI = Windows 등에서 **아이콘·메뉴**를 **마우스**로 선택하여 작업 수행.

## 애플리케이션 배포

### 핵심 개념
- **형상관리(SCM)**: 소프트웨어 개발 과정에서 변경 사항을 체계적으로 관리하는 일련의 활동. 버전 관리, 변경 추적, 릴리스 관리.
- **빌드 자동화 도구**: 컴파일·테스트·패키징을 자동화(Ant·Maven·Gradle·Jenkins·Make).
- **소스코드 검증(Secure Coding)**: 구현 단계의 보안 취약점을 최소화하기 위해 보안 요소를 고려하여 코딩하는 기법.

### 암기 포인트
- ⭐ **형상관리(SCM)** = 소프트웨어 변경 사항을 관리하기 위한 일련의 활동. "개발 과정에서 변경 관리"가 키워드.
- ⭐ 빌드 자동화 도구: **Ant·Maven·Gradle·Jenkins·Make** (IDE 이름이 오면 오답).
- ⭐ **Secure Coding** = 보안 취약점 사전 대응, 안정성·신뢰성 확보를 위한 보안 코딩.
