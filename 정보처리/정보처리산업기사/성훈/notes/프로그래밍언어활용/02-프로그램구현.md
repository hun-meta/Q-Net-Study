## 공통 모듈 구현

### 핵심 개념
- 정의: 결합도(Coupling)와 응집도(Cohesion)를 이해하고 모듈의 독립성을 높이는 설계 원칙을 적용.
- 핵심 포인트: **결합도는 낮게(Loose), 응집도는 높게(High)** = 모듈 독립성 향상. 결합도 순서(강→약): 내용 > 공통 > 외부 > 제어 > 스탬프 > 자료. 응집도 순서(강→약): 기능적 > 순차적 > 교환적 > 절차적 > 시간적 > 논리적 > 우연적.
- 자주 헷갈리는 함정: "가장 강한 결합도"는 **내용 결합도**(다른 모듈 내부 자료 직접 참조). fan-in은 높을수록 재사용성↑, fan-out은 낮을수록 좋음.

### 암기 포인트
- ⭐ 결합도(강→약): 내공외제스자 — **내**용·**공**통·**외**부·**제**어·스탬프·**자**료.
- ⭐ 응집도(강→약): 기순교절시논우 — **기**능적·**순**차적·**교**환적·**절**차적·**시**간적·**논**리적·**우**연적.
- ⭐ 4+1 뷰: 유스케이스(중심), 논리·구현·프로세스·물리.
- ⭐ 공통 모듈 명세: 정확성·명확성·완전성·일관성·추적성.

## 화면 구현

### 암기 포인트
- ⭐ JS 대화상자: `alert()`(경고), `confirm()`(확인/취소), `prompt()`(입력).

- ⭐ CSS: `border-radius`(둥근 테두리), `padding`(내부 여백), `margin`(외부 여백).
- ⭐ JS 배열: `push()`(추가), `pop()`(제거), `length`(길이), `join()`(문자열 변환).

## 서버 프로그램 구현

### 핵심 개념
- 정의: 보안 취약점의 개념, 정보 보안 3대 요소, API와 인터페이스의 이해.
- 핵심 포인트: **정보 보안 3대 요소** = 기밀성(Confidentiality)·무결성(Integrity)·가용성(Availability). 보안 취약점 = 시스템 설계·구현의 문제로 인한 약점. Open API = 누구나 무료로 사용 가능한 공개 API. XSS = 악의적 스크립트를 웹 페이지에 삽입.
- 자주 헷갈리는 함정: 무결성 = "인가된 사용자만 수정 가능"(변조 방지), 기밀성 = "인가된 사용자만 접근 가능"(유출 방지).

### 암기 포인트
- ⭐ 보안 3대 요소: **CIA** — 기밀성(Confidentiality)·무결성(Integrity)·가용성(Availability).
- ⭐ 보안 취약점(Vulnerability): 설계·구현 단계의 문제로 시스템이 가지는 약점.
- ⭐ XSS(Cross-Site Scripting): 악의적 스크립트를 웹 페이지에 삽입해 사용자 정보 유출.
- ⭐ Open API: 공개된 API로 누구나 무료 사용 가능.
