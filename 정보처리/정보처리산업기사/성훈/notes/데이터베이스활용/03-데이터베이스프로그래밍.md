## 데이터 조작 프로시저 작성

> 세세항목: 프로시저 이해 · 프로시저 오류처리(디버깅) 이해.

### 핵심 개념
- **프로시저(Procedure)**: 절차형 SQL을 활용하여 특정 기능을 수행하는 데이터베이스 객체. 일련의 SQL 문을 묶어 실행.
- **디버깅(Debugging)**: 오류가 발생한 코드를 추적하여 수정하는 작업. (프로시저 실행 자체가 아니라, 오류 수정 활동)
- **오류 처리**: 프로시저 내 예외(EXCEPTION) 처리부를 두어 런타임 오류를 관리.

### 암기 포인트
- ⭐ 디버깅 = "오류 코드 추적·수정" (성능 모니터링 도구·절차형 SQL 실행과 구분).
- ⭐ 프로시저 = 절차형 SQL로 특정 기능 수행.

### 기출 연계
- 🔁 기출 2025-2회-필기 #59: 디버깅에 대한 옳은 설명(오류 발생 코드 추적·수정)

## 데이터 조작 프로시저 최적화

> 세세항목: 쿼리 성능 측정 도구(APM 등).

### 핵심 개념
- **APM(Application Performance Management)**: 애플리케이션의 성능 관리를 위해 다양한 모니터링 기능을 제공하는 도구. 느린 쿼리·병목 구간 탐지.
- **최적화 관점**: 실행 계획(Execution Plan) 분석, 인덱스 활용, 불필요한 접근 최소화로 응답시간 개선.
- **쿼리 성능 측정 도구**: APM, 실행 계획(`EXPLAIN`/`DESC`), 프로파일링 도구 등으로 쿼리 처리 비용·소요 시간을 정량 측정.

### 암기 포인트
- ⭐ 성능 관리 모니터링 도구 = **APM**.
