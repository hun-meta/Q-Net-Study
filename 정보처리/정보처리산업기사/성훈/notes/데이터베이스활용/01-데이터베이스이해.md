## 데이터베이스 기초 활용

> 세세항목: 데이터베이스 개념 · 유형 및 특징 · DBMS 필수 기능 · 스키마 3계층 · 데이터 모델 구성요소.

### 핵심 개념
- **DBMS 필수 기능 3가지**: 정의(Definition) · 조작(Manipulation) · 제어(Control). '운영/회복 기능'은 필수 기능이 아님(함정).
- **스키마 3계층**: 외부(사용자 뷰) · 개념(전체 논리 구조, 통합) · 내부(물리 저장 구조). '내용 스키마'는 존재하지 않음.
- **데이터 모델 구성요소 3가지**: 구조(Structure) · 연산(Operation) · 제약조건(Constraint). Structure = 개체 타입과 관계 기술.
- **데이터 모델 유형**: 관계형(릴레이션/2차원 테이블, 가장 널리 사용) · 계층형(트리) · 네트워크형(그래프) · 개체형.

### 암기 포인트
- ⭐ DBMS 필수기능 = **정·조·제** (정의·조작·제어). 운영·회복은 오답.
- ⭐ 스키마 3계층 = **외·개·내**. 개념 스키마 = 조직 전체 통합 논리 구조.
- ⭐ 데이터 모델 구성요소 = **구조·연산·제약조건**.

## 데이터베이스 설계 단계

> 세세항목: 요구조건 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계 순서 및 각 단계 내용.

### 핵심 개념
- **설계 순서**: 요구조건 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계. (가장 먼저 = 요구조건 분석)
- **논리적 설계**: 개념 스키마 평가·정제 → DBMS에 맞는 논리 스키마(테이블) 설계. 트랜잭션 인터페이스 설계는 물리적 설계 단계(함정).
- **물리적 설계**: 파일 저장 구조·액세스 경로 결정, 저장 레코드 양식 설계, 레코드 집중 분석, 접근 경로 설계. 응답시간·저장공간·트랜잭션 처리율 고려.
  - 주의: "개념적 설계 단계의 정보 구조로부터 스키마 생성"은 **논리적 설계** 설명이므로 물리적 설계 항목으로는 틀림.

## 데이터 무결성

> 세세항목: 개체 무결성 · 참조 무결성 · 도메인 무결성 · 널(Null) 값 · 무결성 규정.

### 핵심 개념
- **개체 무결성**: 기본키를 구성하는 어떤 속성도 Null 값이나 중복값을 가질 수 없음. (가장 빈출)
- **참조 무결성**: 외래키 값은 Null이거나 참조 릴레이션의 기본키 값과 동일해야 함.
- **도메인 무결성**: 속성 값이 정의된 도메인에 속한 값이어야 함.
- **널(Null) 값**: 아직 알려지지 않았거나 모르는 값, "해당 없음" 등 정보 부재를 나타내는 특수한 데이터 값.
- **무결성 규정**: 불법 사용자 보호는 '보안'의 영역이지 무결성이 아님(함정).

### 암기 포인트
- ⭐ **개체 무결성 = 기본키 NOT NULL & NOT 중복**. 이 문장 그대로 출제됨.
- ⭐ Null = "모르는/해당 없음" 값 (원자값·참조값·무결값 아님).

## 엔티티 및 릴레이션

> 세세항목: 릴레이션 구성·특성 · 키의 종류 · 도메인 · 차수(Degree)·카디널리티(Cardinality).

### 핵심 개념
- **릴레이션 특성**: 튜플 중복 불가, 튜플/속성 순서 무의미, 모든 속성 값은 원자값, 속성명은 유일. ("한 릴레이션 내 튜플이 모두 같다"는 틀림)
- **차수(Degree) = 속성(열) 수 / 카디널리티(Cardinality) = 튜플(행) 수**.
- **도메인**: 하나의 속성이 취할 수 있는 모든 원자값들의 집합.
- **키 종류**: 슈퍼키(유일성O 최소성X) → 후보키(유일성O 최소성O) → 기본키(후보키 중 대표) / 대체키(기본키 제외 후보키) / 외래키(다른 릴레이션 기본키 참조). '보조키=후보키 중 대표'는 틀림(그건 기본키).

### 암기 포인트
- ⭐ **차수=열, 카디널리티=행** (D-C 순서 헷갈림 주의).
- ⭐ 슈퍼키는 최소성 X, 후보키는 최소성 O. 기본키 = 후보키 중 대표.

## 자료구조의 이해

> 세세항목: 스택 · 큐 · 데크 · 리스트 · 트리 · 그래프 · 정렬 · 해싱. (산업기사 DB에 자료구조가 다수 출제됨)

### 핵심 개념
- **스택(LIFO)**: 한쪽 끝에서 삽입·삭제, 후입선출. 응용=서브루틴 호출·인터럽트 처리·수식계산. (작업 스케줄링은 아님)
- **큐(FIFO)**: 한쪽 삽입·다른 쪽 삭제, 선입선출. **데크(Deque)**: 양쪽 끝 삽입·삭제(입력제한/출력제한 데크).
- **선형 자료구조**: 리스트·스택·큐·데크. **비선형**: 트리·그래프. (그래프는 선형 아님)
- **트리 순회**: 전위(Pre)=루트→좌→우, 중위(In)=좌→루트→우, 후위(Post)=좌→우→루트. 차수(Degree)=노드의 최대 자식 수.
- **그래프**: n정점 무방향 그래프 최대 간선 수 = n(n-1)/2.
- **정렬**: 버블·삽입·선택 등. 회전(Pass)별 중간 결과 계산이 빈출.
- **해싱**: Collision(충돌)=서로 다른 레코드가 동일 주소 갖는 현상, Synonym=충돌로 같은 Home Address 갖는 레코드 집합. 폴딩법=키를 여러 부분으로 분해해 합/XOR.
- **연결 리스트**: 삽입·삭제 쉽지만 임의 검색은 느림(포인터 추가 공간 필요).

### 암기 포인트
- ⭐ 스택=LIFO, 큐=FIFO, 데크=양쪽. 스케줄링은 큐/스택 아님.
- ⭐ 전위=루트 먼저, 후위=루트 나중. 무방향 그래프 최대 간선 = n(n-1)/2.
- ⭐ Collision=충돌 현상, Synonym=동일주소 레코드 집합.

## E-R 모델링 이해

## 정규화 이해

> 세세항목: 정규화 단계(1NF~BCNF) · 이상(Anomaly) 현상 · 함수 종속성.

### 핵심 개념
- **이상(Anomaly)**: 정규화 미수행 시 데이터 중복으로 발생. 종류 = 삽입·삭제·갱신 이상. ('조회/검색 이상'은 아님)
- **정규화 단계**:
  - 비정규 → **1NF**: 원자값(반복그룹 제거)
  - 1NF → **2NF**: 부분 함수 종속 제거 (복합키 일부에 종속)
  - 2NF → **3NF**: 이행적 함수 종속 제거 (A→B, B→C 일 때 A→C)
  - 3NF → **BCNF**: 모든 결정자가 후보키가 되도록 분해

### 암기 포인트
- ⭐ **1NF=원자값, 2NF=부분종속제거, 3NF=이행종속제거, BCNF=결정자 후보키**.
- ⭐ 이상 종류 = 삽입·삭제·갱신 (조회 아님).

### 기출 연계

## 관계 데이터 연산

> 세세항목: 관계대수·관계해석 · 일반 집합 연산자·순수 관계 연산자 · 카티션 프로덕트.

### 핵심 개념
- **관계대수**: 절차적(연산 순서 명시). **관계해석**: 비절차적(원하는 정보만 정의, 프레디킷 해석 기반). 둘은 처리 능력이 동등.
- 함정: "관계대수가 비절차적이다"는 틀림(관계대수는 절차적).
- **일반 집합 연산자**: ∪(합집합) · ∩(교집합) · −(차집합) · ×(카티션 프로덕트=교차곱).
- **순수 관계 연산자**: ⋈(조인) · σ(셀렉트) · π(프로젝트) · ÷(디비전). (⋈는 일반 집합 연산자 아님)
- **카티션 프로덕트**: 결과 카디널리티 = 두 릴레이션 튜플 수의 곱(예: 4×5=20).

### 암기 포인트
- ⭐ 관계대수=절차적, 관계해석=비절차적. ⋈(조인)은 순수 관계 연산자.
- ⭐ 카티션프로덕트 결과 = 행 수의 곱.

### 기출 연계

## 물리 데이터베이스 설계

> 세세항목: 물리적 설계 · 시스템 카탈로그(데이터 사전) · 파일 구조(직접 파일).

### 핵심 개념
- **물리적 설계**: 논리 구조 → 물리 저장장치에 저장 가능한 구조 변환. 파일 저장 구조·액세스 경로 결정, 응답시간·저장공간·트랜잭션 처리율 고려. ("개념 설계 정보 구조로부터 스키마 생성"은 논리적 설계이므로 물리 설계 항목으로는 틀림; "스키마 평가·정제"도 논리적 설계)
- **시스템 카탈로그(=데이터 사전)**: DBMS가 자체 생성·유지하는 시스템 데이터베이스, 메타데이터 저장(기본 테이블·뷰·인덱스·접근권한 등). 일반 사용자는 **검색 가능**하지만 직접 INSERT/UPDATE/DELETE 갱신은 불가(함정: "검색할 수 없다"/"직접 갱신할 수 있다"는 틀림).
- **직접 파일**: 해시 함수/색인 테이블로 키→주소 변환해 저장.

### 암기 포인트
- ⭐ 시스템 카탈로그 = 데이터 사전 = 메타데이터. 사용자 검색 O, 직접 갱신 X.
- ⭐ 물리 설계 vs 논리 설계 구분: 스키마 생성·평가정제 = 논리, 저장구조·액세스경로 = 물리.
