# 01. 응용SW기초기술활용

> 출처: `_공통/출제기준/정보시스템기반기술.md`의 주요항목 순번을 따른다.

## 운영체제 기초 활용

### 핵심 개념
- 정의: 프로세스 스케줄링 — 실행 대기 중인 여러 프로세스 가운데 어떤 것에 CPU를 먼저 배정할지 순서를 정하는 작업. (프로세스 = 현재 실행 중인 프로그램)
- 왜 중요한가: 스케줄링 알고리즘을 선점형/비선점형으로 분류해 짝짓는 문제가 기출 단골.

### 상세 정리

**선점형 vs 비선점형**

- **비선점형(Non-preemptive)**: 한 프로세스가 CPU를 차지하면 그 작업이 끝날 때까지 다른 프로세스가 CPU를 강제로 뺏을 수 없는 방식. 동기/비동기와는 다른 개념 — "진행 중인 작업을 다 끝내야만 다음 작업으로 넘어간다"는 뜻.
- **선점형(Preemptive)**: 실행 도중이라도 운영체제가 CPU를 강제로 회수해 다른 프로세스에 줄 수 있는 방식.

| 방식 | 선점 여부 | 동작 규칙 |
|------|----------|-----------|
| FIFO (FCFS) | 비선점 | 먼저 도착한(First In) 프로세스부터 처리 |
| SJF (Shortest Job First) | 비선점 | CPU가 비었을 때 대기열에서 실행시간이 가장 짧은 작업을 골라 끝까지 실행 |
| SRT (Shortest Remaining Time) | 선점 | 새 프로세스 도착 시 "남은 시간"을 비교해 더 짧은 쪽에 CPU를 뺏어줌 |
| Round-Robin | 선점 | 모든 프로세스가 Time Quantum(시간 할당량)만큼만 번갈아 사용, 시간 초과 시 강제 교체 |

**SJF가 비선점형인 이유 (헷갈림 주의)**

- "가장 짧은 것부터"라는 규칙은 **CPU가 비어 있을 때(현재 작업이 완전히 끝났을 때)만** 적용된다.
- 실행 도중에 더 짧은 작업이 도착해도 절대 뺏기지 않고 하던 작업을 끝낸 뒤, 그때 대기열에서 최단 작업을 고른다. → 선택 "타이밍"이 시작 시점뿐이므로 비선점형.

**SRT의 '남은 시간' 의미**

- 여러 작업을 동시에 시작해서 남은 시간을 보는 게 아니라, **현재 실행 중인 프로세스가 끝날 때까지 남은 시간**을 뜻한다.
- 새 프로세스가 도착할 때마다 「실행 중 작업의 잔여 시간」 vs 「새 작업의 소요 시간」을 비교해 더 짧은 쪽으로 CPU를 즉시 넘긴다(선점).
- 예: A(10분짜리)를 2분 진행해 잔여 8분인 시점에 B(3분짜리) 도착 → SJF는 A를 마저 끝내지만, SRT는 A를 중단하고 B를 먼저 실행.

### 암기 포인트
- ⭐ 비선점 = **SJF, FIFO** / 선점 = **SRT, Round-Robin**. 이름에 R이 들어간 SRT·RR은 "뺏는(선점)" 쪽으로 기억.
- ⭐ SJF와 SRT의 차이는 "언제 고르느냐": 끝나고 고르면 SJF(비선점), 도착할 때마다 다시 재면 SRT(선점).
