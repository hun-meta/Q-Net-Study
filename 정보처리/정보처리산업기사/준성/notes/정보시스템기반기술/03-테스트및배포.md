# 03. 테스트및배포

> 출처: `_공통/출제기준/정보시스템기반기술.md`의 주요항목 순번을 따른다.

## 애플리케이션 테스트 수행

### 핵심 개념
- 정의: 블랙박스 테스트 — 소프트웨어의 내부 구조나 작동 원리를 보지 않고, 겉으로 드러나는 기능과 인터페이스(입력→출력)에만 초점을 맞춰 검사하는 테스트 방식. 내부를 알 수 없는 '검은 상자'처럼 취급한다.
- 왜 중요한가: 블랙박스/화이트박스의 특징과 각 기법의 소속을 바꿔치기해서 "옳지 않은 것"을 고르게 하는 문제가 기출 단골.

### 상세 정리

**블랙박스 테스트의 특징**

- 사용자가 특정 값을 **입력**했을 때 기대하는 **출력**(결과)이 정확히 나오는지를 확인한다.
- 프로그램의 **내부 구조를 고려하지 않는다**.
- **소프트웨어 인터페이스**에서 실시된다.
- 각 **기능이 완전히 작동되는 것을 입증**하는 데 목적이 있다 (기능 테스트).

**블랙박스 vs 화이트박스**

| 구분 | 블랙박스 테스트 | 화이트박스 테스트 |
|------|----------------|------------------|
| 관점 | 기능·인터페이스 (입력→출력) | 내부 구조·소스 코드 |
| 내부 구조 | 고려하지 않음 | 직접 확인함 |
| 대표 기법 | 동치 분할, 경계값 분석, 원인-효과 그래프 등 | **기초 경로 검사, 제어 구조 검사** 등 |

- 함정 포인트: **기초 경로 검사·제어 구조 검사는 화이트박스 테스트의 종류**다. 블랙박스 종류로 제시되면 틀린 보기.

### 암기 포인트
- ⭐ "경로·구조·조건·루프"처럼 **코드 내부를 들여다보는 이름이 붙은 기법은 전부 화이트박스**. 블랙박스는 입력·출력 값 중심(동치, 경계값).

### 기출 연계
- 🔁 기출 2025-1회-필기 #2: 블랙박스 테스트 설명 중 옳지 않은 것 → 기초 경로 검사·제어 구조 검사는 화이트박스 기법 (정답 ④)

## UI 테스트

### 핵심 개념
- 정의: 사용자 인터페이스(UI)의 4대 기본 원칙 — **직관성, 유효성, 학습성, 유연성**. UI가 갖춰야 할 근본 설계 원칙 4가지.
- 왜 중요한가: 4대 원칙에 **없는 성질(명확성·일관성 등)을 원칙인 것처럼 끼워 넣고** "옳지 않은 것"을 고르게 하는 유형이 기출로 출제됨.

### 상세 정리

**UI 4대 기본 원칙**

| 원칙 | 정의 |
|------|------|
| 직관성 | 누구나 쉽게 이해하고 사용할 수 있어야 한다 |
| 유효성 | 사용자의 목적을 정확하고 완벽하게 달성해야 한다 |
| 학습성 | 누구나 쉽게 배우고 익힐 수 있어야 한다 |
| 유연성 | 사용자의 요구사항을 최대한 수용하고 오류를 최소화해야 한다 |

**직관 vs 명확 — 왜 '명확성'은 기본 원칙이 아닌가**

- **직관(直觀)** = "척 보면 아는 것". 설명을 읽거나 머리로 따져보지 않아도 경험·감각으로 **보는 순간 즉시** 사용법을 아는 것. (예: 돋보기 아이콘 → 검색, 휴지통 아이콘 → 삭제)
- **명확(明確)** = "뜻이 뚜렷해 오해가 없는 것". 의미는 정확히 전달되지만, 인지하려면 **글을 읽고 이해하는 과정**이 필요할 수 있다. (예: "선택하신 파일을 영구적으로 삭제하시겠습니까?"라는 문구는 명확하지만 직관적 디자인은 아님)
- "**개념적으로** 쉽게 인지하도록 설계한다"는 말은 시스템의 구조·원리를 머리로 논리적으로 이해해야 한다는 뜻에 가깝다. 좋은 UI는 개념 이해를 요구하는 게 아니라 **생각할 필요 없이 바로 쓰게** 만드는 것(직관성)이 핵심이다.

### 암기 포인트
- ⭐ UI 4대 원칙: **직관성·유효성·학습성·유연성** — '명확성'이 보기로 나오면 원칙이 아니므로 틀린 설명.
- ⭐ UI는 '개념적으로 이해시키는' 것이 아니라 '**직관적으로 바로 쓰게**' 하는 것.

### 기출 연계
- 🔁 기출 2025-1회-필기 #6: UI 기본 원칙 설명 중 옳지 않은 것 → '명확성'은 4대 기본 원칙이 아님 (정답 ③)
