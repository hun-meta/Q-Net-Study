# 03. 테스트및배포

> 출처: `_공통/출제기준/정보시스템기반기술.md`의 주요항목 순번을 따른다.

## 애플리케이션 테스트 수행

### 핵심 개념
- 정의: 블랙박스 테스트 — 소프트웨어의 내부 구조나 작동 원리를 보지 않고, 겉으로 드러나는 기능과 인터페이스(입력→출력)에만 초점을 맞춰 검사하는 테스트 방식. 내부를 알 수 없는 '검은 상자'처럼 취급한다.
- 왜 중요한가: 블랙박스/화이트박스의 특징과 각 기법의 소속을 바꿔치기해서 "옳지 않은 것"을 고르게 하는 문제가 기출 단골.

### 상세 정리

**블랙박스 테스트의 특징**

- 사용자가 특정 값을 **입력**했을 때 기대하는 **출력**(결과)이 정확히 나오는지를 확인한다.
- 프로그램의 **내부 구조를 고려하지 않는다**.
- **소프트웨어 인터페이스**에서 실시된다.
- 각 **기능이 완전히 작동되는 것을 입증**하는 데 목적이 있다 (기능 테스트).

**블랙박스 vs 화이트박스**

| 구분 | 블랙박스 테스트 | 화이트박스 테스트 |
|------|----------------|------------------|
| 관점 | 기능·인터페이스 (입력→출력) | 내부 구조·소스 코드 |
| 내부 구조 | 고려하지 않음 | 직접 확인함 |
| 대표 기법 | 동치 분할, 경계값 분석, 원인-효과 그래프 등 | **기초 경로 검사, 제어 구조 검사** 등 |

- 함정 포인트: **기초 경로 검사·제어 구조 검사는 화이트박스 테스트의 종류**다. 블랙박스 종류로 제시되면 틀린 보기.

### 암기 포인트
- ⭐ "경로·구조·조건·루프"처럼 **코드 내부를 들여다보는 이름이 붙은 기법은 전부 화이트박스**. 블랙박스는 입력·출력 값 중심(동치, 경계값).

### 기출 연계
- 🔁 기출 2025-1회-필기 #2: 블랙박스 테스트 설명 중 옳지 않은 것 → 기초 경로 검사·제어 구조 검사는 화이트박스 기법 (정답 ④)
