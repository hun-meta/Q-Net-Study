# 02. 통합구현

## 모듈 구현

### 핵심 개념
- **모듈(Module)**: 소프트웨어 구조를 이루며 다른 것과 구별될 수 있는 **독립적 기능 단위**. 하나~몇 개의 논리적 기능을 수행하는 명령어 집합. 모여서 완전한 프로그램 구성.
- **효과적 모듈 설계(결합도↓·응집도↑)**: 모듈 간 **결합도(Coupling)는 낮게(약하게)**, 모듈 내 **응집도(Cohesion)는 높게(강하게)**. 모듈 크기는 적당히 작게, 기능적 독립성 유지.
- **재사용·재공학**: 소프트웨어 **재사용(Reuse)**=기존 모듈/컴포넌트를 다시 활용(개발비용·기간 단축, 품질 향상). **재공학(Reengineering)**=기존 시스템을 분석(역공학)해 개선·재구축(재개발 대비 비용·위험 감소, 품질 향상). 단, 새 기술 도입 한계·성능 한계는 재공학의 단점.
- **소프트웨어 공학 일반**: 공학적으로 잘된 SW는 유지보수성·신뢰성·효율성·사용성 확보. 공학의 기본 원칙 = 품질 높이고, 비용·기간 단축, 지속적 개선.

### 암기 포인트
- ⭐ 모듈 = **독립적 기능 단위**. 좋은 모듈 = **결합도↓(Loose), 응집도↑(High)**.
- ⭐ 재공학 vs 재개발: 재공학이 **비용·위험 감소** (전부 새로 짜지 않음).
- ⭐ "위험 분석" 키워드면 나선형(SDLC) — 01-요구사항확인 참조.

## 통합 구현 관리

### 핵심 개념
- **IDE(통합개발환경)**: 편집·컴파일·디버깅·빌드 등 개발에 필요한 도구를 하나로 통합한 환경(Eclipse, IntelliJ, VS Code 등). 코드 작성부터 실행·테스트까지 통합 지원.
- **형상 관리(Configuration Management)**: 소프트웨어 개발 전 과정에서 산출물(소스코드·문서·빌드 등)의 **변경 사항을 추적·통제**하는 활동. 관리 항목: 소스코드, 설계문서, 테스트 케이스, 빌드/릴리스 산출물 등. 절차: 식별→통제→상태보고→감사. (형상관리 도구: Git, SVN 등)
- **위험 관리(Risk Management)**: 프로젝트에 내재된 위험 요소를 인식하고 영향을 분석해 사전 예측·대비하는 활동. **위험 모니터링**=식별된 위험의 상태를 지속 추적·관리.

### 암기 포인트
- ⭐ 형상관리 = **변경사항 추적·통제**(버전 관리의 상위 개념). 관리 항목에 '하드웨어 사양서' 등 비SW 항목은 제외될 수 있음(오답 유도).
- ⭐ 위험 관리 = 위험 **인식→분석→대비→모니터링**.
