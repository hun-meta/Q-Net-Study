# 01. 데이터입출력구현

> 출처: `_공통/출제기준/소프트웨어개발.md`의 주요항목 순번을 따른다.

## 논리 데이터저장소 확인

### 핵심 개념
- **자료구조 분류**: 선형구조(배열, 연결리스트, 스택, 큐, 데크) / 비선형구조(트리, 그래프). 트리는 사이클이 없는 계층형 비선형 구조.
- **스택(Stack)**: LIFO(후입선출). 한 쪽 끝에서 삽입(Push)/삭제(Pop). 후위표기 계산, 괄호검사, 되돌리기(Ctrl+Z), 함수호출 관리에 사용. 순서 A,B,C,D를 Push 후 Pop하면 D,C,B,A(역순)가 되지만 Push/Pop이 섞이면 다양한 출력 가능 → 불가능한 출력 판별이 단골.
- **트리 순회**: 전위(Preorder=Root→L→R), 중위(Inorder=L→Root→R), 후위(Postorder=L→R→Root). 차수(Degree)=한 노드가 갖는 자식 수의 최댓값. 단말(Terminal) 노드=자식이 없는 노드.
- **수식 표기법 변환**: 중위(Infix) → 전위/후위 변환과 후위식 연산결과 계산은 스택을 이용. (스택 응용 단골)
- **해싱(Hashing)**: 키(key)를 해시함수로 변환해 저장위치(홈 주소)를 산출. 함수 종류: 제산법(나머지), 제곱법, 폴딩법(나눠서 더함/XOR), 기수변환법, 무작위법.
- **반정규화(Denormalization)**: 정규화된 데이터베이스의 성능 향상을 위해 **중복·통합·분리**를 허용하는 기법. 유형: 테이블 통합/분할/추가, 중복컬럼 추가 등.
- **DBMS 필수기능**: 정의기능(스키마/구조·타입·제약조건 명시), 조작기능(CRUD), 제어기능(무결성·보안·권한).

### 암기 포인트
- ⭐ 스택=**LIFO**(역순 출력), 큐=**FIFO**. 트리 순회는 **전(루트앞)·중(루트가운데)·후(루트끝)**.
- ⭐ 편향 트리(한쪽으로 치우친 트리) = **최악의 검색 효율**. 균형 트리가 좋음.
- ⭐ 방향 그래프 최대 간선 = n(n−1), 무방향 그래프 최대 간선 = n(n−1)/2.
- ⭐ 반정규화 = 성능↑ 위해 **의도적 중복**(정규화 완화).

## 물리 데이터저장소 설계

### 핵심 개념
- **물리 데이터저장소 설계**: 논리 데이터 모델(E-R)을 실제 저장 구조(테이블·컬럼·데이터타입)로 변환 + **인덱스·파티션·물리적 저장 구조**를 설계하는 단계.
- **파티션(Partition) 설계**: 대용량 테이블을 작은 논리 단위로 분할해 조회 성능↑·관리 용이. 유형: **범위(Range)·해시(Hash)·목록(List)·조합(Composite)** 분할.
- **인덱스(Index) 설계**: 검색 성능 향상을 위한 **B-Tree 기반** 접근 경로 생성. 조회↑·DML↓ 트레이드오프 고려, **선택도가 낮은(카디널리티 높은)** 컬럼에 적합.
- **ORM/트랜잭션 인터페이스**: ORM 프레임워크(Hibernate 등)로 객체-관계 매핑, **트랜잭션 인터페이스**로 ACID(원자성·일관성) 보장.

### 암기 포인트
- ⭐ 파티션 4유형 = **범위·해시·목록·조합**. (Range/Hash/List/Composite)

## 데이터 조작 프로시저 작성

> 세부항목: 프로시저 · 프로그램 디버깅 · 단위테스트 도구. 본 과목 기출에서 직접 출제된 문항은 적음(관련 문항은 04 애플리케이션테스트관리 참조).

### 핵심 개념
- **프로시저(Procedure)**: SQL 문장들을 묶어 하나의 이름으로 저장·반복 실행하는 프로그램 단위. 절차적 SQL(**PL/SQL** 등)로 작성. **트리거(Trigger)**는 특정 이벤트(DML 등) 발생 시 자동 실행되는 특수 프로시저.
- **프로그램 디버깅(Debugging)**: 오류(버그)를 찾아 수정. **중단점(Breakpoint)·변수 추적(Watch)·스텝 실행** 등 활용. 구문·논리·실행 오류를 단계별 해결.
- **단위테스트 도구**: 프로시저·모듈별 개별 테스트(JUnit·DBUnit 등). 단위 모듈의 정상 처리·예외 처리를 검증(→ 04 통합테스트로 확장).

## 데이터 조작 프로시저 최적화

> 세부항목: 쿼리 성능 측정 · 소스코드 인스펙션. 쿼리 성능/인스펙션 관련 문항은 적음(정적 분석·인스펙션은 04 애플리케이션 성능 개선 참조).

### 핵심 개념
- **쿼리 성능 측정·최적화**: **실행계획(Execution Plan)** 분석으로 접근 경로·비용 확인 → 느린 쿼리 개선. **인덱스 활용·조인 순서 최적화·Full Table Scan 회피**가 핵심.
- **소스코드 인스펙션**: 코드를 수동/도구로 검토해 결함을 사전 발견. **정적 분석**(실행 없이 코드 품질·보안 점검)과 연관 → 04 애플리케이션 성능 개선.
- **최적화 기법**: 중복 SQL 제거, **바인드 변수**(하드파싱 회피), 필요 컬럼만 SELECT, 정렬·그룹화·조건절 최소화.
