## 인터페이스 설계 확인

### 핵심 개념
- **인터페이스**: 이기종 시스템·컴포넌트 간 데이터를 주고받는 **연계(통신) 규약·창구**. 설계 시 연계 대상 시스템·데이터 흐름·주기·표준(포맷) 확인.
- **EAI(Enterprise Application Integration)**: 기업 내 **이기종 애플리케이션을 통합**해 데이터를 연계·통신. 구축 유형:
  - **Point-to-Point(점대점)**: 애플리케이션 1:1 직접 연결(가장 단순, 확장성↓).
  - **Hub & Spoke(허브앤스포크)**: 중앙 허브가 모든 연계 통제(관리 편리, 허브 병목/장애 시 전체 영향).
  - **Message Bus(메시지 버스)**: 버스(미들웨어) 기반 느슨한 결합.
  - **Hybrid(하이브리드)**: Hub & Spoke + Message Bus 결합(그룹 내 버스, 그룹 간 허브).
- **데이터 표준/포맷**: 인터페이스 통신용 데이터 포맷 — **JSON**(속성-값 쌍, 경량), **XML**, CSV 등.

### 암기 포인트
- ⭐ EAI 4유형 = **점대점·허브앤스포크·메시지버스·하이브리드**.
- ⭐ Hybrid = **허브+버스 결합**(그룹 내 버스, 그룹 간 허브).
- ⭐ JSON = **속성-값 쌍(Attribute-Value Pairs)** 개방형 표준 포맷.

## 인터페이스 기능 구현

### 핵심 개념
- **인터페이스 보안**: 통신 구간을 보호. **네트워크 영역 솔루션**: **IPSec/SSL·TLS**(전송 구간 암호화), **VPN**, 방화벽, **WAF**(웹 방화벽). (IPSec, SSL/TLS, VPN이 보안 솔루션; 로드밸런서·NAT 등은 트래픽 분산이지 보안 솔루션 아님 — 오답 유도)
- **소프트웨어 연계 기술**: **AJAX**(JavaScript 비동기 통신, 클라이언트-서버 간 XML 데이터 송수신), **JSON/XML**(데이터 교환 포맷), **REST/RESTful API**, **웹 서비스(SOAP)**, **API 게이트웨이**.

### 암기 포인트
- ⭐ 네트워크 보안 = **IPSec / SSL·TLS / VPN**(구간 암호화). 로드밸런서는 보안 솔루션 아님.
- ⭐ AJAX = **비동기 통신**(클라이언트-서버, XML 교환).

## 인터페이스 구현 검증

### 핵심 개념
- **인터페이스 구현 검증**: 연계 기능이 정상 동작하는지 확인. **인터페이스 구현 검증 도구**: **xUnit**(JUnit, pytest 등 단위 테스트 프레임워크), **STAF**(Software Testing Automation Framework), **FitNesse**, **Selenium**, **NTA(Network Traffic Analyzer)**.
  - xUnit, STAF, Selenium, FitNesse, NTA 등이 검증 도구; 일반 빌드/형상 도구는 아님(오답 유도).
- **설계 산출물/인터페이스 명세서**: 인터페이스 명세서(연계 대상·데이터·주기·포맷·오류 처리)를 기준으로 구현 검증.

### 암기 포인트
- ⭐ 인터페이스 검증 도구 = **xUnit / STAF / Selenium / FitNesse / NTA**.
