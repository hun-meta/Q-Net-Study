## 제품소프트웨어 패키징

### 핵심 개념
- **패키징**: 개발 완료된 프로그램을 배포·설치 가능한 형태로 묶는 작업. 모듈·환경·사용자 OS 고려. **패키징 도구 활용 고려사항**: 암호화·보안(DRM), 사용자 편의성(UI/매뉴얼), 표준 준수, 다양한 환경(이기종/모바일) 지원, 설치·제거 편의성.
- **DRM(디지털 저작권 관리, Digital Rights Management)**: 디지털 콘텐츠의 **무단 복제·배포를 방지**하고 저작권 보호. 데이터 암호화·키 관리·권한 제어로 사용 제한.
  - **구성 요소**: 콘텐츠 제공자, 콘텐츠 분배자, 콘텐츠 소비자, DRM 컨트롤러(권한·사용 제어), 클리어링 하우스(권한·과금 처리), 패키저.
  - **기술 요소**: 암호화, 키 관리, 암호화 파일 생성, 식별 기술, 저작권 표현, 정책 관리, 크랙 방지, 인증.

### 암기 포인트
- ⭐ DRM = **무단 복제 방지·저작권 보호**. 핵심 구성 = **클리어링 하우스(과금/권한)**, **DRM 컨트롤러(사용 제어)**.
- ⭐ 패키징 고려사항 오답 유형: "운영체제 고려 안 함" 등은 틀린 설명(이기종 환경 고려 필수).

## 제품소프트웨어 매뉴얼 작성

### 핵심 개념
- **설치 매뉴얼**: 제품 SW 설치 과정을 안내하는 문서. 포함 항목: 제품 개요, 설치 환경(하드웨어/소프트웨어), 설치 절차(순서도), 설치 시 주의/오류 처리, 제거(Uninstall) 방법, 사용자 등록, 고객 지원(연락처). 목차·절차는 사용자(비전문가) 관점으로 작성.
- **국제 표준 제품 품질 특성(ISO/IEC 9126 / 25000)**: SW 품질을 6가지 특성으로 정의 — **기능성(Functionality), 신뢰성(Reliability), 사용성(Usability), 효율성(Efficiency), 유지보수성(Maintainability), 이식성(Portability)**.
  - **기능성 하위 특성**: 적합성, 정확성, 상호운용성, 보안성, 준수성.
- **소프트웨어 품질 목표**: 개발자 관점 항목 = 유지보수성, 이식성, 시험용이성(Testability), 변경용이성 등. 사용자 관점 = 신뢰성, 사용성, 효율성.

### 암기 포인트
- ⭐ 품질 6특성 = **기신사효유이** (기능성·신뢰성·사용성·효율성·유지보수성·이식성).
- ⭐ 이식성(Portability) = 다른 하드웨어/환경으로 **쉽게 이동·수정**되는 능력.
- ⭐ ISO/IEC **25000**(SQuaRE)이 9126을 대체하는 최신 표준. 기능성 하위에 보안성·정확성·상호운용성 포함.

## 제품소프트웨어 버전관리

### 핵심 개념
- **버전 관리(Version Control)**: 소프트웨어 산출물의 변경 이력을 추적·관리. 동시 수정 충돌 방지, 변경 내용 추적, 이전 버전 복원 지원.
  - **용어**: 체크아웃(저장소에서 작업 복사본 가져오기), 체크인/커밋(수정 사항을 저장소에 갱신), 브랜치, 머지.
  - **방식**: **집중형(Centralized, SVN)**=중앙 서버 하나에 저장. **분산형(Distributed, Git)**=로컬에 전체 저장소 복제, 오프라인 작업 가능.
- **빌드 자동화 도구**: 소스 컴파일·테스트·배포를 자동화. **Gradle**(그루비 기반, 안드로이드 기본), **Maven**(XML/POM), **Jenkins**(CI 서버, 스케줄링·지속적 통합), **Ant**(XML).

### 암기 포인트
- ⭐ 버전관리 = 변경 **추적·복원·충돌 방지**. 분산형=**Git**(로컬 저장소), 집중형=**SVN**(중앙 서버).
- ⭐ 빌드 자동화: **Gradle**(안드로이드), **Jenkins**(CI). Maven은 XML 기반.
