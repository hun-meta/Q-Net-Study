# 01. 서버프로그램구현

> 출처: `_공통/출제기준/프로그래밍언어활용.md`의 주요항목 순번을 따른다.

## 개발환경 구축

### 핵심 개념
- 정의: 응용 소프트웨어 개발을 위해 **하드웨어·소프트웨어(IDE·프레임워크·빌드도구)·네트워크** 환경을 구성하는 작업. 서버 개발 프레임워크(Spring·Django·Node.js 등)로 일관된 구조를 제공한다.
- 핵심 구성: **웹 서버**(정적 콘텐츠, Apache·Nginx) vs **WAS**(동적 처리, Tomcat) 구분이 자주 출제. **빌드 도구**(Ant·Maven·Gradle)와 **형상관리**(Git·SVN)를 구별할 것.
- 함정: Kerberos는 빌드 도구가 **아님**(인증 프로토콜). Eclipse는 하드웨어가 아닌 소프트웨어(IDE).

### 기출 연계

- 🔁 기출 2024-2회-필기 #61: 하드웨어 환경이 아닌 것 찾기 — 개발환경 구성요소 분류(Eclipse)
- 🔁 기출 2025-2회-필기 #79: 프레임워크(Framework) 개념 설명으로 옳은 것
- 🔁 기출 2025-3회-필기 #67: 빌드 도구(Ant/Maven/Gradle)와 보안 프로토콜(Kerberos) 구분

## 공통 모듈 구현

### 핵심 개념
- 정의: 여러 프로그램에서 공통으로 사용할 수 있는 **모듈**을 구현하는 것. **재사용·모듈화**로 중복을 줄이고 유지보수성을 높인다.
- 결합도(Coupling) — 모듈 간 의존성. **낮을수록 좋음**(약→강): 자료(Data) → 스탬프(Stamp) → 제어(Control) → 외부(External) → 공통(Common) → 내용(Content). 제어 결합은 '처리 절차 통제·권리전도(플래그)'가 특징.
- 응집도(Cohesion) — 모듈 내 기능 집중도. **높을수록 좋음**: 기능적 > 순차적 > 교환(통신)적 > 절차적 > 시간적 > 논리적 > **우연적(가장 낮음)**.
- 모듈화: 모듈 수 ↑ → 모듈당 크기 ↓, **Fan-In**(상위 호출 수, 높게) / **Fan-Out**(하위 호출 수, 낮게) 설계 지침.

### 기출 연계
- 🔁 기출 2023-2회-필기 #76: 결합도(Coupling) 단계 약→강 순서 정렬
- 🔁 기출 2023-3회-필기 #64: 결합도 낮은→높은 순 정렬 (자료<스탬프<제어<외부<공통<내용)
- 🔁 기출 2023-3회-필기 #74: 우연적 응집(Coincidental) — 연관성 없는 기능들의 집합
- 🔁 기출 2023-3회-필기 #77: 모듈화 특징 (모듈 수 증가 시 크기는 작아짐, 틀린 설명)
- 🔁 기출 2024-1회-필기 #77: Control Coupling — 권리전도현상 발생 결합도 식별
- 🔁 기출 2024-2회-필기 #64: 처리절차 통제·권리전도 결합도 — 제어 결합도 식별
- 🔁 기출 2024-2회-필기 #74: 모듈 F의 Fan-In/Fan-Out 계산 — 호출 관계 분석(3/2)

- 🔁 기출 2025-1회-필기 #62: 모듈 결합도 약화 원칙, 틀린 설명 찾기
- 🔁 기출 2025-1회-필기 #78: 모듈 내부 기능 참조 = 내용 결합도(Content Coupling)

## 서버 프로그램 구현

### 핵심 개념
- 정의: 서버 측 비즈니스 로직을 구현하고 **보안 취약성을 식별**·제거하는 과정. API(특히 RESTful API) 설계와 MVC 패턴이 핵심.
- 보안 취약점 대표: **버퍼 오버플로(Buffer Overflow)** = 메모리 취급 오류로 발생; 그 외 SQL Injection·XSS·CSRF(XSRF).
- 함정: 메모리 할당 오류·입력 검증 부재로 발생하는 취약점 → **버퍼 오버플로**.

### 기출 연계
- 🔁 기출 2023-2회-필기 #78: 메모리 취약점 — 버퍼 오버플로 식별
- 🔁 기출 2023-3회-필기 #62: 메모리 취급 오류로 발생하는 취약점 — 버퍼 오버플로 식별

## 배치 프로그램 구현

### 핵심 개념
- 정의: 사용자 개입 없이 **일괄적·자동적**으로 대량 데이터를 처리하는 프로그램. (예: 정산·마감)
- 필수 요소 4가지: **자동화**(Auto)·**안정성**(Reliability)·**대용량 처리**(Performance)·**무결성**(Integrity). 사용자 상호작용은 요소가 아님.
- 유형: **정기 배치**(주기적), **이벤트성 배치**(특정 조건 충족 시 수행), **온디맨드 배치**(요청 시).

### 기출 연계
- 🔁 기출 2023-2회-필기 #79: 배치 프로그램 필수 요소 설명 중 틀린 것
- 🔁 기출 2024-2회-필기 #73: 조건 충족 시 수행되는 배치 — 이벤트성 배치
- 🔁 기출 2025-2회-필기 #73: 특정 조건 충족 시 수행되는 이벤트성 배치 식별
- 🔁 기출 2025-3회-필기 #76: 배치 프로그램 필수 요소(자동화·안정성·대용량·무결성) 설명
