# 02. IT프로젝트정보시스템구축관리

> 출처: `_공통/출제기준/정보시스템구축관리.md`의 주요항목 순번을 따른다.

## 네트워크 구축 관리

### 핵심 개념
- **정의**: IT 신기술(프로토콜·통신 기술) 동향을 파악하고 네트워크 장비(라우터·스위치)를 선정·구축·관리.
- **IT 신기술·네트워크 프로토콜**: **MQTT**(IoT 발행·구독 경량 메시징), **Mesh**(대규모 디바이스 그물망 직접 접속), **SDN**(제어부·데이터부 분리), **NFC**(10cm 이내 근접), **USN**(RFID 태그 무선 수집), **CSMA/CA**(무선랜 충돌 회피 대기), **RIP**(최대 홉수 15).
- **네트워크 장비**: 라우터(L3 경로 설정), 백본 스위치; 프로토콜-포트(HTTP 80·SSH 22·Telnet 23 등), DPI(전 계층 패킷 콘텐츠 분석).
- **함정**: **CSMA/CA**(무선, 충돌 *회피*) vs **CSMA/CD**(유선, 충돌 *검출*) 혼동 주의; **RIP** 도달 한계 홉수 = **15**(16=불가).

## SW 구축 관리

### 핵심 개념
- **정의**: IT 신기술·SW 개발 트렌드를 파악하고 구축 단계에서 SW 개발보안 정책을 수립·적용.
- **IT 신기술·SW 개발 트렌드**: **하둡**(빅데이터 분산 컴퓨팅), **TensorFlow**(머신러닝 오픈소스), **Docker**(컨테이너 배포 자동화), **디지털 트윈**(물리 자산 가상화), **BaaS**(블록체인 클라우드 서비스), **PaaS-TA**(국내 개방형 클라우드 플랫폼).
- **SW 개발보안 정책**: 신기술 도입 시 보안 요구사항(Secure SDLC)을 요구사항 정의에 반영·추적.
- **함정**: **PaaS-TA** = *국내* 개방형 클라우드 플랫폼(글로벌X); **Docker** = 컨테이너(가상머신X) 기반 배포 자동화.

## HW 구축 관리

### 핵심 개념
- **정의**: IT 신기술·서버장비 동향을 반영해 스토리지·고가용성·보안 운영 체계를 구축·관리.
- **서버장비 운영**: **SAN**(광채널 스위치 기반 고속 공유 스토리지), NAS·DAS(스토리지 구분), **고가용성**(HACMP—공유 디스크 클러스터링), **Secure-OS**(TCB 기반 보안 커널).
- **IT 신기술·서버**: **SDDC**(소프트웨어 정의 데이터센터), **SDS**(스토리지 가상화), **HSM**(하드웨어 보안 키 모듈).
- **함정**: **SAN** = *광채널(Fiber Channel)* 스위치 기반(파일 단위 공유는 NAS); **SDS**(스토리지) vs **SDDC**(데이터센터 전체) vs **SDN**(네트워크) 구분.

- 🔁 기출 2023-2회-필기 #97: SAN — 광채널 스위치 기반 공유 스토리지

- 🔁 기출 2024-1회-필기 #82: 고가용성 솔루션(HACMP) — 공유 디스크 클러스터링

## DB 구축 관리

### 핵심 개념
- **정의**: DB 기술 트렌드·관리기능·표준화를 이해하고 데이터베이스 구축 단계에서 적용·관리.
- **데이터베이스 기술 트렌드**: 빅데이터 분석(상관분석), **Sqoop**(하둡·RDBMS 간 데이터 전송).
- **DB 관리·표준화**: **메타데이터**(데이터를 설명하는 데이터), **데이터 다이어트**(데이터 축소·중복 제거), **병행수행 문제**(갱신분실·모순성·연쇄복귀 등) → 병행제어로 해결.
- **함정**: **Sqoop** = 하둡↔*RDBMS* 전송 도구(하둡 내부 처리X); 병행제어의 핵심 목적 = **갱신분실** 방지·**일관성** 보장.

### 기출 연계

**DB 기술 트렌드·관리기능·표준화 (Sqoop·상관분석·메타데이터·데이터다이어트·병행수행)**

- 🔁 기출 2024-2회-필기 #98: 데이터 다이어트(Data Diet) 설명
