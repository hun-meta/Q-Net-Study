# 03. 소프트웨어개발보안구축

> 출처: `_공통/출제기준/정보시스템구축관리.md`의 주요항목 순번을 따른다.

## SW개발 보안 설계

### 핵심 개념
- **Secure SDLC**: 보안을 SDLC 전 단계에 통합. 방법론 = **Seven Touchpoints**·CLASP·SDL·MS SDL. **OWASP** = 웹 보안 취약점 연구 비영리 단체.
- **정보보안 3요소**: **기밀성**(인가자만 접근) · **무결성**(인가자만 수정) · **가용성**(필요 시 접근 가능); 추가로 **부인방지**(송수신 증거).
- **입력데이터 검증 및 표현**: **SQL Injection**(악의적 SQL 삽입), **XSS**(웹페이지 악의적 스크립트 실행) → 입력값 검증·이스케이프로 방어.

## SW개발 보안 구현

### 핵심 개념
- **암호 알고리즘**: **대칭키**(암호화·복호화 키 동일—AES·DES, 빠름) / **비대칭키**(공개키·개인키—**RSA** 소인수분해) / **해시**(단방향—무결성). **Salt** = 동일 패스워드를 다른 값으로 저장.
- **코드오류**: **널 포인터 역참조**(Null 체크), **Stack Guard/카나리**(복귀주소 변조—버퍼오버플로우 탐지), 종료되지 않는 반복문·재귀함수.
- **캡슐화**: Public 메소드가 Private 배열을 그대로 반환하면 외부에서 배열 조작 가능 → 방어적 복사 필요.

### 기출 연계

**암호 알고리즘 (대칭·비대칭·해시·RSA·Salt·스트림)**

- 🔁 기출 2023-3회-필기 #93: 소인수분해 기반 공개키 암호(RSA)
