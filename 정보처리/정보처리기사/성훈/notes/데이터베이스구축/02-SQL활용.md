## 기본 SQL 작성 — DDL

### 핵심 개념
- **DDL(Data Definition Language)**: 데이터 정의어. `CREATE`(생성), `ALTER`(변경), `DROP`(삭제).
- `CREATE TABLE`: 속성 타입 지정, NOT NULL 여부, 기본키 지정, CHECK 제약조건 정의 가능. 단, **속성 타입 "변경"은 ALTER의 기능**이므로 CREATE TABLE에는 포함되지 않는다.
- `CREATE`는 스키마, 도메인, 인덱스 등을 정의할 때도 사용.

### 암기 포인트
- ⭐ DDL = **CAD**(Create, Alter, Drop). UPDATE·DELETE는 DML.
- ⭐ CREATE TABLE 기능: 타입 지정·NOT NULL·기본키·CHECK. 타입 **변경**은 불가.

## 기본 SQL 작성 — 트랜잭션

### 핵심 개념
- **트랜잭션(Transaction)**: 데이터베이스에서 **하나의 논리적 기능을 수행하는 작업의 단위**. ACID 특성: **원자성(Atomicity)**, 일관성(Consistency), **고립성(Isolation)**, 지속성(Durability).
- **상태**: Active → **Partially Committed**(최종 연산 직후, DB 반영 전) → Committed / Aborted.
- **연산**: `COMMIT`(반영 확정), `ROLLBACK`(원상 복귀). `Recovery` = 장애로 손상된 DB를 정상 상태로 복구.
- **병행제어**: 다중 트랜잭션의 동시 실행 제어. 기법: **로킹(Locking)**, 타임스탬프, 다중 버전 기법. (시분할 기법은 아님)
- **로킹 단위**: 한꺼번에 잠글 수 있는 객체의 크기. 단위가 **작으면** 오버헤드 증가·**병행성(공유도) 증가**. 단위가 **크면** 오버헤드 감소·**병행성 감소**.
- **회복 기법**: 즉각 갱신 기법은 로그(log)를 필요로 함.

### 암기 포인트
- ⭐ ACID = **원자성**(All or Nothing) / 일관성 / **고립성**(Isolation, 끼어들기 불가) / 지속성.
- ⭐ Partially Committed = 연산 끝났지만 **DB 반영 전**.
- ⭐ **로킹 단위 반비례**: 작을수록 오버헤드↑ 병행성↑, 클수록 오버헤드↓ 병행성↓.
- ⭐ 병행제어 기법 = 로킹·타임스탬프·다중버전 (**시분할은 아님**).
- ⭐ COMMIT/ROLLBACK → **원자성** 보장.

## 기본 SQL 작성 — 데이터 사전

### 핵심 개념
- **데이터 사전(시스템 카탈로그)**: DBMS가 스스로 생성·유지하는 특별한 테이블 집합. **메타데이터**(데이터에 대한 데이터)를 저장.
- 데이터 객체(테이블, 인덱스, 뷰 등)의 정의·명세 정보를 유지 관리.
- **일반 사용자는 SQL로 검색만 가능, 직접 생성·수정·갱신할 수 없음**(DBMS가 자동 관리).
- 데이터 디렉토리: 데이터 사전의 데이터에 실제 접근하는 데 필요한 위치 정보를 관리.

### 암기 포인트
- ⭐ 데이터 사전 = 시스템 카탈로그 = **메타데이터 저장소**.
- ⭐ 사용자가 **직접 갱신 불가** (DBMS가 자동 관리) → 이것이 핵심 오답 패턴.

### 기출 연계

## 고급 SQL 작성 — 뷰

### 핵심 개념
- **뷰(View)**: 하나 이상의 기본 테이블로부터 유도된 **가상 테이블**. `CREATE VIEW`로 정의, `DROP VIEW`로 삭제.
- **물리적으로 존재하지 않음**(저장장치에 실제 데이터 없음). 독자적인 인덱스를 가질 수 없음.
- 장점: 논리적 독립성 제공, 데이터 보안 용이, 사용자 데이터 관리 용이.
- **제약**: 삽입·갱신·삭제 연산에 제약이 따름. 뷰 위에 또 다른 뷰 정의 가능.
- `DROP VIEW 뷰명 CASCADE`: 의존하는 뷰까지 연쇄 삭제.
- 뷰 정의는 `ALTER`로 변경 불가(삭제 후 재생성).

### 암기 포인트
- ⭐ 뷰 = **가상 테이블**, 물리적 존재 X, **인덱스 없음**.
- ⭐ 삽입·갱신·삭제 시 **제약 있음** (제약이 없다는 설명은 틀림).
- ⭐ `DROP VIEW ... CASCADE` = 의존 뷰까지 **모두 삭제**.

## 고급 SQL 작성 — 집합 연산자

### 핵심 개념
- **UNION**(합집합, 중복 제거), **UNION ALL**(합집합, **중복 허용**), INTERSECT(교집합), MINUS/EXCEPT(차집합).
- `UNION ALL`: 양쪽 SELECT 결과를 모두 합쳐 반환(중복 행도 그대로 유지).

### 암기 포인트
- ⭐ **UNION ALL** = 중복을 **제거하지 않음**. UNION은 중복 제거.

### 기출 연계

## 고급 SQL 작성 — 서브쿼리

### 핵심 개념
- **서브쿼리(Subquery)**: 다른 SQL문 내부에 중첩된 SELECT문. WHERE절, HAVING절 등에 위치.
- 예: `SELECT 이름 FROM 직원 WHERE 팀코드=(SELECT 팀코드 FROM 직원 WHERE 이름='정도일');`

### 암기 포인트
- ⭐ 서브쿼리 = **쿼리 안의 쿼리**, 괄호로 감싸서 사용.

### 기출 연계
- 🔁 기출 2024-1회-필기 #58: 서브쿼리 — WHERE 팀코드=(SELECT 팀코드 FROM 직원 WHERE 이름='정도일')
