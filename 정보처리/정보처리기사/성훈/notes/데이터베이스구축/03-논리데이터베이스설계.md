# 03. 논리데이터베이스설계

> 출처: `_공통/출제기준/데이터베이스구축.md`의 주요항목 순번을 따른다.
> 세부항목: 관계데이터베이스 모델(관계 데이터 모델·관계대수·관계해석) · 데이터모델링 및 설계(데이터모델 개념·E-R 모델·논리적 데이터모델링·정규화)

## 관계데이터베이스 모델 — 관계 데이터 모델

### 핵심 개념
- **릴레이션(Relation)**: 테이블. 행=**튜플(Tuple)**, 열=**속성(Attribute)**.
- **차수(Degree)** = 속성(열)의 수. **카디널리티(Cardinality)** = 튜플(행)의 수.
- **도메인(Domain)**: 하나의 속성이 가질 수 있는 **원자값들의 집합**.
- 릴레이션의 특징: ① 모든 튜플은 서로 다른 값(유일성), ② 속성은 유일한 이름, ③ **튜플 사이 순서 없음**, ④ 속성 사이 순서 없음, ⑤ 모든 속성 값은 **원자값**.
- 가능한 튜플 최대 수 = 각 속성 도메인 크기의 **곱**.

### 암기 포인트
- ⭐ **차수(Degree)** = 열 수(속성 수), **카디널리티(Cardinality)** = 행 수(튜플 수).
- ⭐ **도메인** = 속성이 가질 수 있는 원자값의 집합.
- ⭐ 튜플 순서 **없음**, 속성 순서 **없음**, 모든 값은 **원자값** — "순서가 있다"는 오답 패턴.
- ⭐ 속성의 수 = **Degree** (Cardinality가 아님).

## 관계데이터베이스 모델 — 관계대수

### 핵심 개념
- **관계대수(Relational Algebra)**: 릴레이션을 조작하기 위한 연산의 집합. **절차적(procedural) 언어** — 수행할 연산의 순서를 명시.
- **일반 집합 연산자**: 합집합(∪), 교집합(∩), **차집합(−)**, **카티션 프로덕트(×)**.
- **순수 관계 연산자**: **Select(σ)**, **Project(π)**, Join(▷), Division(÷).
- **Select(σ)**: 조건 만족 튜플 추출(수평적 부분집합). **Project(π)**: 특정 속성 추출(수직적 부분집합).
- **카티션 프로덕트(R×S)**: 차수 = R차수 + S차수, 카디널리티 = R카디널리티 × S카디널리티.
- 관계대수식 → SQL: `π이름(σ학과='교육'(학생))` = `SELECT 이름 FROM 학생 WHERE 학과='교육';`

### 암기 포인트
- ⭐ 관계대수 = **절차적**, 관계해석 = **비절차적** ("비절차적 언어"라는 설명은 관계대수에 대해 틀림).
- ⭐ 순수 관계 연산자 = **σ Select, π Project, ▷ Join, ÷ Division**.
- ⭐ 일반 집합 연산자 = **∪ ∩ − ×**(차집합·카티션 프로덕트는 순수 관계 연산자가 **아님**).
- ⭐ 카티션 프로덕트: **차수는 합, 카디널리티는 곱**.

## 관계데이터베이스 모델 — 관계해석

### 핵심 개념
- **관계해석(Relational Calculus)**: 원하는 릴레이션을 정의하는 방법을 제공. **비절차적(non-procedural) 언어** — 어떻게 유도하는지가 아닌 '무엇을 원하는지' 기술.
- 논리 기호: **∀(For All, 모든 것에 대하여)**, **∃(There Exists, 존재한다)**.

### 암기 포인트
- ⭐ ∀ = **"모든 것에 대하여"**(For All). ∃ = "존재한다"(There Exists).
- ⭐ 관계해석 = **비절차적** / 관계대수 = **절차적**.

## 데이터모델링 및 설계 — 데이터모델 개념

### 핵심 개념
- **데이터 모델 구성 요소**: **구조(Structure)**, **연산(Operation)**, **제약조건(Constraint)**. (관계는 구성 요소가 아님)
- **스키마(Schema)** 3계층: **외부 스키마**(사용자 관점), **개념 스키마**(조직 전체의 논리적 구조, DB 전체 정의), **내부 스키마**(물리적 저장 구조).
- **DBMS 필수 기능**: Definition(정의), **Manipulation(조작)**, Control(제어).
- **데이터베이스의 특성**: 실시간 접근성, 계속적인 변화, 동시 공용, 내용에 의한 참조.
- **데이터 모델 표시 요소**: 논리적 데이터 구조, 연산, 제약조건 (출력 구조는 아님).

### 암기 포인트
- ⭐ 데이터 모델 구성 요소 = **구조·연산·제약조건** (Operation, Constraint, Structure).
- ⭐ 개념 스키마 = **조직 전체의 DB 구조** (데이터 개체·관계·제약조건·접근권한·무결성 규칙 명세).
- ⭐ DBMS 기능 = **정의·조작·제어**(Definition·Manipulation·Control).

## 데이터모델링 및 설계 — 개체-관계(E-R) 모델

### 핵심 개념
- **E-R 모델**: 개체 타입과 관계 타입을 기본 요소로 현실 세계를 개념적으로 표현.
- **E-R 다이어그램 기호**: **사각형**(개체 타입), **마름모**(관계 타입), **타원**(속성), **선**(연결).
- 오너-멤버(Owner-Member) 관계는 E-R 모델이 아닌 **네트워크 모델**의 개념.

### 암기 포인트
- ⭐ E-R 기호 = **사각형(개체) / 마름모(관계) / 타원(속성) / 선(연결)**.
- ⭐ 속성 = **타원**(오각형·삼각형이 아님). 연결 = **선**(삼각형이 아님).
- ⭐ 오너-멤버 관계 = 네트워크 모델 (E-R 모델이 아님).

### 기출 연계

## 데이터모델링 및 설계 — 논리적 데이터모델링

### 핵심 개념
- **데이터베이스 설계 단계**: 요구조건분석 → **개념적 설계**(E-R Diagram, DBMS 독립적 개념 스키마) → **논리적 설계**(목표 DBMS에 **독립적** 논리 스키마, 정규화) → **물리적 설계** → 구현(DDL 작성).
- 개념적 설계 산출물 = **E-R Diagram**. 트랜잭션 인터페이스 설계·작성은 **구현 단계**(또는 요구조건분석)의 작업이지 개념적 설계가 아님.
- 논리적 설계 = 목표 DBMS에 **독립적**(종속적이 아님) 논리 스키마 설계.

### 암기 포인트
- ⭐ 설계 순서: **요구분석 → 개념적 → 논리적 → 물리적 → 구현**.
- ⭐ 개념적 설계 산출물 = **E-R Diagram**, DBMS **독립적**.
- ⭐ 논리적 설계 = DBMS **독립적** 논리 스키마 (종속적이라는 설명은 틀림).

## 데이터모델링 및 설계 — 데이터베이스 정규화

### 핵심 개념
- **정규화(Normalization)**: 데이터 중복과 이상(Anomaly) 현상을 줄이기 위해 릴레이션을 분해하는 과정. 데이터 구조의 안정성 최대화가 목적.
- **이상(Anomaly) 현상 3가지**: **삽입 이상**, **삭제 이상**, **갱신 이상**. (검색 이상·종속 이상은 아님)
- **정규형 단계**:
  - **제1정규형(1NF)**: 모든 도메인이 **원자값**(반복 그룹 분해).
  - **제2정규형(2NF)**: 1NF + **부분 함수 종속 제거**(완전 함수 종속).
  - **제3정규형(3NF)**: 2NF + **이행적 함수 종속 제거**.
  - **BCNF**: 3NF + **결정자가 후보키가 아닌 함수 종속 제거**(모든 결정자가 후보키).
- **이행 규칙**: X→Y이고 Y→Z이면 X→Z (이행적 함수 종속).
- 정규화는 논리적 설계 단계에서 수행 (개념적 설계 이전이 아님).

### 암기 포인트
- ⭐ 이상 현상 = **삽입·삭제·갱신** (검색·종속 이상은 아님).
- ⭐ 정규화 단계: **1NF**(원자값) → **2NF**(부분 종속 제거) → **3NF**(이행 종속 제거) → **BCNF**(결정자=후보키).
- ⭐ **3NF → BCNF** 조건 = **결정자이면서 후보키가 아닌 것 제거** (가장 빈출).
- ⭐ 이행 규칙 = X→Y, Y→Z이면 **X→Z**.
- ⭐ 정규화 = 종속성을 **분해**(통합이 아님). 중복 데이터 **활성화**(X).

### 기출 연계
