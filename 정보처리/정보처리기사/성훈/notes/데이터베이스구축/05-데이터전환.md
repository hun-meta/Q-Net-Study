# 05. 데이터전환

> 출처: `_공통/출제기준/데이터베이스구축.md`의 주요항목 순번을 따른다.
> 세부항목: 데이터 전환 기술(초기데이터 구축·ETL·파일 처리) · 데이터 전환 수행 · 데이터 정제

## 데이터 전환 기술

### 핵심 개념
- **데이터 전환(Data Migration)**: 기존 시스템의 데이터를 새 시스템으로 이관하는 작업.
- **ETL(Extraction·Transformation·Loading)**: 데이터를 **추출·변환·적재**하는 데이터 웨어하우스 구축 핵심 기술.
- **데이터 웨어하우스 & OLAP**: ETL로 구축된 데이터 웨어하우스에서 **OLAP(On-Line Analytical Processing)** 로 다차원 분석.
- **OLAP 기본 연산**: Roll-Up, Drill-Down, Slice, Dice, Pivot. (Translate는 OLAP 연산이 아님)

### 암기 포인트
- ⭐ ETL = **추출·변환·적재**(Extraction·Transformation·Loading).
- ⭐ OLAP 연산 = **Roll-up·Drill-down·Slice·Dice·Pivot** (Translate는 아님).

### 기출 연계
- 🔁 기출 2023-3회-필기 #60: OLAP 기본 연산이 아닌 것(Translate)

## 데이터 전환 수행

### 핵심 개념
- **데이터 전환 수행 계획**: 전환 대상·일정·절차·책임자를 정의한 계획서. 전환 전 사전 점검 및 **체크리스트(Checklist)** 작성 필수.
- **데이터 검증**: 전환 후 원본·목적 시스템 간 **데이터 건수·합계·내역 일치** 여부를 점검하여 누락·오류 데이터 확인.
- **체크리스트 점검 항목**: 전환 완료 여부, 데이터 정합성, 오류 발생 건수, 보완 작업 이력 등.

### 암기 포인트
- ⭐ 데이터 전환 수행 = **계획 수립 → 체크리스트 → 데이터 검증** 순서.
- ⭐ 데이터 검증 = 원본과 목적 시스템 간 **정합성·건수·합계** 확인.

## 데이터 정제

### 핵심 개념
- **데이터 정제**: 전환 대상 데이터의 오류(중복·누락·형식 위반)를 식별·수정하는 작업. 데이터 품질 확보가 목적.
- **데이터 품질 분석**: 정확성·완전성·일관성·유일성·적시성 등 품질 기준에 따라 데이터 상태 평가.
- **오류 데이터 측정**: 오류 건수·오류율을 정량 측정하여 정제 우선순위를 결정하고, 정제 후 재검증으로 품질 개선 효과 확인.

### 암기 포인트
- ⭐ 데이터 정제 = 오류 데이터 식별·수정 / 목적 = 데이터 품질 확보.
- ⭐ 오류 데이터 측정 = 오류 건수·오류율 **정량화**하여 우선순위 결정.
