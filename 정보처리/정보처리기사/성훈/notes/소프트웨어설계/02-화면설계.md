## UI 요구사항 확인

### 핵심 개념
- **UI(사용자 인터페이스)**: 사용자와 시스템이 상호작용하는 창구. CLI·GUI·NUI·OUI로 분류된다.
- **UI 설계 원칙(4대)**: 직관성(누구나 쉽게 이해·사용) · 유효성(정확한 목적 달성) · 학습성(쉽게 배움) · 유연성(오류 최소화·다양한 상호작용 수용).
- **UI 설계 지침**: 일관성(버튼·조작법 통일) · 주요 기능 메인 노출 · 치명적 오류는 적극적으로 사용자에게 알림 · 다양한 계층(직무·연령·성별) 수용. "조작 방법을 다양하게"·"오류를 숨긴다"는 모두 틀린 설명.
- **UI 종류**: CLI(명령 문자열 입력, DOS·UNIX) · GUI(그래픽) · NUI(멀티터치·제스처 등 자연스러운 움직임 인식, Pinch·Press·Flick; Flow는 제외) · OUI(유기적).

### 암기 포인트
- ⭐ 직관성 = "쉽게 이해하고 사용"이 핵심 키워드. 문제에 이 문구 → 정답 직관성.
- ⭐ NUI 제스처 = **Pinch·Press·Flick**(Flow는 해당 없음). CLI는 "정해진 명령 문자열 입력".
- ⭐ 오류 메시지 지침: 소리·색 적극 활용(텍스트만 전달은 틀림), 부정적 내용도 적극 알림.

## UI 설계

### 핵심 개념
- **UI 설계 도구(산출물)**: 와이어프레임(레이아웃 스케치) · **목업(Mockup)**: 실제 화면과 유사한 **정적 모형**(배치만, 구현 X) · **스토리보드(Storyboard)**: 와이어프레임 + **흐름·설명·기능 명세**(가장 많이 쓰임) · **프로토타입(Prototype)**: **동적** 인터랙션 가능 모형 · **유스케이스**: 사용자 요구사항(목표 달성) 기술.
- **감성공학(Ergonomics)**: 인간의 **감성·지각·행동** 특성을 제품 설계에 반영해 사용자 만족을 높이는 학문. 감성 이미지·단어 분석·행동 관찰 기법 활용.
- **UI 설계 절차**: UI 요구사항을 반영해 **와이어프레임 → 스토리보드 → 프로토타입** 순으로 점진적 구체화. **사용성(Usability) 테스트**로 피드백 수렴 후 최종 화면 확정.

### 암기 포인트
- ⭐ **목업 = 정적 모형**(배치만, 구현 X) vs **프로토타입 = 동적 모형**(인터랙션 가능). 이 구분이 출제 포인트.
- ⭐ 스토리보드 = 와이어프레임에 설명·흐름·기능을 덧붙인 종합 산출물.
