## 인터페이스 요구사항 확인

### 핵심 개념
- **요구공학(Requirements Engineering)**: 무엇을 개발해야 하는지 요구사항을 **정의·분석·명세·관리**하는 체계적 프로세스(학문). 사용자 요구의 복잡성·잦은 변경을 통제해 **품질↑·프로젝트 실패↓**가 목적.
- **요구사항 계층**: **요구공학 ⊃ 요구사항 개발**(도출→분석→명세→확인). "요구공학은 요구사항 개발의 한 요소"라는 설명은 **역전 오답**.
- **내외부 인터페이스 요구사항**: 시스템 **내부** 컴포넌트 간 + **외부** 이기종 시스템 간 연계(데이터·규격·주기·보안) 요구사항을 식별·명세화.

### 암기 포인트
- ⭐ 요구공학 = 요구사항 **정의·분석·관리**를 다루는 학문. 요구사항 개발을 포함하는 더 큰 개념.

## 인터페이스 대상 식별

### 핵심 개념
- **인터페이스 시스템**: 내·외부 시스템 간 데이터를 주고받는 **연계 대상**. 송신·수신 시스템, 연계 컴포넌트, 미들웨어 솔루션을 식별.
- **인터페이스 대상 식별 절차**: 시스템 아키텍처(구성도) 분석 → 송·수신 시스템·컴포넌트·솔루션 추출 → **데이터 흐름·연계 주기·규격** 명세화.
- **FEP(Front-End Processor, 전단처리기)**: 입력 데이터를 메인 프로세서가 처리하기 **전에 미리 전처리**해 프로세서 부담·처리 시간을 줄여주는 프로그램·하드웨어(통신·인터페이스 전단 담당).

### 암기 포인트
- ⭐ FEP = "입력 데이터를 **미리 처리**해 프로세서 부담 ↓". 통신·인터페이스 전단 처리 역할.

## 인터페이스 상세 설계

### 핵심 개념
- **미들웨어(Middleware)**: 클라이언트와 서버(또는 운영체제와 응용 프로그램) 사이에서 **통신·데이터 교환·관리**를 담당하는 시스템 소프트웨어. 위치 투명성(Location Transparency) 제공, 재사용 가능한 서비스 구현, 컴포넌트 연결 인프라 제공.
- **미들웨어 종류**: DB 미들웨어 · **MOM**(메시지 지향 - 비동기·메시지 큐, 느리고 안정적 응답에 적합) · **TP-Monitor**(트랜잭션 감시·제어) · **WAS**(웹 애플리케이션 서버 - **동적** 콘텐츠·웹 환경, WebLogic·WebSphere) · RPC · ORB.
- **송수신 데이터 처리 방식**: 실시간 방식(즉시) · **배치 방식**(대량 데이터 일괄 처리) · 지연 처리 방식.

### 암기 포인트
- ⭐ 미들웨어는 "애플리케이션과 **사용자** 사이에서만"이라는 설명은 틀림 → **서버·컴포넌트·OS 사이**까지 아우름. 내부 동작을 사용자가 쉽게 확인해야 한다는 설명도 틀림(투명성).
- ⭐ **WAS = 동적 콘텐츠** 처리. "정적 콘텐츠 처리용"이라는 설명은 틀림(정적은 웹 서버).
- ⭐ **MOM = 비동기·메시지 큐**, "즉각적 응답이 필요한 온라인 업무에 적합"이라는 설명은 틀림. 대량 데이터 = **배치 방식**.
