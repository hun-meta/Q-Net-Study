## 성훈 (2026-07-21)

**주제: 프로토타입(Prototype) 모형 — 소프트웨어 생명주기(SDLC) 개발 모형**

- **개념**: 개발 초기에 핵심 기능·인터페이스 중심의 **견본품(시제품)**을 빠르게 만들어 사용자 피드백으로 요구사항을 계속 정제하는 개발 모형. 상위 개념은 **SDLC(소프트웨어 생명주기) 모형**이고, 형제 개념으로 폭포수·나선형·애자일·V-모형이 있다.
- **핵심 포인트(선택지 함정 대비)**:
  - 프로토타입의 1차 목적은 **요구사항 명확화·검증**과 **사용자–개발자 의사소통 갭 해소**다. "성능 최적화"나 "최종 산출물 완성"이 목적이라는 선지는 오답.
  - 유형은 **폐기형(Throwaway)**(요구 수집 후 버림) / **진화형(Evolutionary)**(보완해 최종 시스템으로 발전) 두 가지.
  - **위험 분석(Risk Analysis)**을 반복하는 건 프로토타입이 아니라 **나선형(Spiral)**이다(혼동 주의).
- **비교 요약**: 순차·변경취약 → **폭포수** / 위험 분석 반복 → **나선형** / 테스트 단계 1:1 매핑 → **V-모형** / 스프린트 반복의 경량 방법론 → **애자일**.
- **왜 틀렸나(내 메모)**: 개념은 알았지만 형제 모형(특히 나선형의 위험 분석)과 헷갈려 찍음(X). → 노트 `성훈/notes/소프트웨어설계/01-요구사항확인.md`의 SDLC 비교표로 복습.
